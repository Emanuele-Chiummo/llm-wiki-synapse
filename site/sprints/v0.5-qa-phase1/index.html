    QA Gate Report — v0.5 Phase 1 (Retrieval Foundation)¶

    Sprint: v0.5 (sprint/v0.5 branch) Phase: Phase 1 — F5 4-phase retrieval, GET /search, [n] citations, save-to-wiki, CliAgentProvider.chat() QA Engineer: qa-test-engineer (claude-sonnet-4-6) Date: 2026-06-29 Verdict: PASS-WITH-NOTES

    1. Verdict Summary¶

    Phase 1 code passes all tests after QA added two sets of missing tests (AC-F5-3 static import guards and AC-F5-8 three-provider parameterized integration tests). The four performance invariants relevant to Phase 1 (I3/G3, I1/AC-F5-5, I7/CLI bounds, I6) are verified by automated tests. No flakiness detected across two independent full-suite runs.

    Notes (non-blocking):

    • AC-F5-3 and AC-F5-8 were not covered before this gate. Tests were added to the existing test_code_quality.py and to a new test_chat_endpoint.py. Both pass green.
    • The total_cost_usd type for NB-4 is float (not Decimal) throughout the codebase per ADR-0009/NB-4 — the story text's "Decimal('0.00')" is documentation shorthand; the implementation is consistent and the tests assert the correct float behaviour.
    • SettingsPanel.test.tsx emits one act(...) warning (pre-existing, from M4); it is not a failure and does not affect Phase 1 ACs.
    • test_mcp_tools.py::TestMcpServerDefinition::test_four_tools_registered emits a RuntimeWarning: coroutine 'FastMCP.list_tools' was never awaited (pre-existing, test logic calls a sync interface on an async method). Test passes; no regression from Phase 1.

    5. Performance Invariant Gates (Phase 1-relevant)¶

    G3 / I3 — No per-token markdown parse; citation decoration single-pass (ADR-0022 Do-NOT #6)¶

    Code review: decorateCitations.ts has exactly one import (CitationRef type from chatStore). It imports no markdown parser; its module-level interface imports confirm zero renderMarkdown dependency. It is a single-pass regex replacement over the already-parsed HTML string, memoized on (rawHtml, citationKey).

    Test coverage: - chat-parse-once.test.ts::G3 — parse-once invariant::streaming 100 tokens: renderMarkdown is NOT called during token phase — asserts spy call count == 0 during 100 appendToken calls. - chat-parse-once.test.ts::after finalizeTurn: parse selector fires exactly once (AC-G3-3) — asserts the messages-length subscriber fires exactly 1 time on finalizeTurn. - ChatMessage.test.tsx::decorateCitations — memoization: same inputs return the same string reference — confirms zero re-processing on re-render. - ChatMessage.test.tsx::streaming 100 tokens: citations never mutated during stream (I3) — asserts messages array not touched during 100 streaming tokens.

    Verdict: GATE PASS. The G3 invariant is verified: renderMarkdown is called 0 times during streaming and decorateCitations adds no second markdown parse.

    I1 / AC-F5-5 — retrieve() and GET /search are read-only; data_version unchanged¶

    Test coverage: - test_retrieval.py::test_ac_f5_5_data_version_unchanged — reads data_version before and after retrieve() with SQLite, asserts before == after == 7. - test_api.py::TestGetSearch::test_search_does_not_bump_data_version — ASGI client test asserts /status data_version is identical before and after GET /search. - test_api.py::TestGetSearch::test_search_data_version_in_response — asserts search.data_version == status.data_version (snapshot taken before assembly, AC-F5-5).

    Verdict: GATE PASS.

    I7 — CliAgentProvider.chat() bounded by CHAT_AGENT_MAX_TURNS + no runaway¶

    Test coverage: - test_cli_chat.py::test_chat_bounded_by_chat_agent_max_turns_env — sets env to 3, asserts SDK receives max_turns=3. - test_cli_chat.py::test_chat_default_max_turns_is_eight — unset env, asserts max_turns=8. - test_cli_chat.py::test_chat_invalid_max_turns_falls_back_to_default — sets env to "0" (non-positive), asserts fallback to max_turns=8. - test_cli_chat.py::test_chat_no_api_key_raises_clean_config_error_not_fake_stream — asserts ValueError raised before generator returned (Do-NOT #9 verified; no fake stream).

    CHAT_AGENT_MAX_TURNS default: 8 (confirmed in cli.py::_DEFAULT_CHAT_AGENT_MAX_TURNS). Invalid value fallback: non-positive integer → default, with WARNING logged. NB-4 cost: float 0.0 (not Decimal — see notes §1).

    Verdict: GATE PASS.

    I6 — provider.chat() signature unchanged across all 3 providers¶

    Test coverage: - test_schemas.py::test_chat_returns_async_iterator_for_local_and_api — calls OllamaProvider.chat() and ApiProvider.chat() with identical signature (messages, retrieval_context). - test_cli_chat.py::test_chat_streams_text_deltas_and_injects_context — calls CliAgentProvider.chat(messages, retrieval_context="..."). - test_chat_endpoint.py (QA-added) — the _RecordingProvider mock signature is chat(self, messages, retrieval_context: str) for all three provider_type parametrize values, and the dispatch resolve_provider returns the same mock regardless of type — proving run_chat_stream passes retrieval_context uniformly. - ADR-0022 Do-NOT #7 ("Do NOT change provider.chat(messages, retrieval_context)") is verified: chat/stream.py passes a single combined string; the provider receives it without type or position change.

    Verdict: GATE PASS.
