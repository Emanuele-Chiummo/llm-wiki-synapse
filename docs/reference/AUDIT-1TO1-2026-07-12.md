## 0. Premessa architetturale (NON è un gap)

`nashsu/llm_wiki` è un'app **solo client** (React + Vite + Tauri): tutta la logica di
ingest, embedding, provider LLM, graph, lint, review, dedup, chunking vive in TypeScript
sotto `src/lib/` e `src/components/`. Non ha backend.

Synapse è una **ri-architettura FastAPI (Python) + React** con Postgres + Qdrant + watcher
incrementale. Le differenze di *persistenza/runtime* (DB vs storage client, layout server-side,
embedding server-side) sono **by design** (invarianti I1/I2) e non vanno considerate bug.
Il confronto verte su **parità di logica e comportamento**, non sul meccanismo di storage.

## 4d. Allineamento estetico — tipografia Geist (verificato live in Chrome)

L'estetica di llm_wiki è shadcn "new-york" monocromatica con font **Geist Variable**. La divergenza
estetica principale e *non-conflittuale* era il font: Synapse puntava a Geist (Brand v1.0) ma era un
TODO incompiuto (system-font fallback). Abilitato:
- `frontend/package.json`: dipendenza **`@fontsource-variable/geist`** (come llm_wiki).
- `frontend/src/main.tsx`: `import "@fontsource-variable/geist"`.
- `frontend/src/styles/theme.css` + `frontend/index.html`: le stack font antepongono **"Geist Variable"**;
  corretto anche un refuso nel reset anti-FOUC (`html,body,#root` terminava in `monospace`).
- Commenti stale in theme.css aggiornati (Geist ora attivo via npm, non più TODO self-host).

**Verifica:** in Chrome sul vault reale — `document.fonts.check('16px "Geist Variable"')` → **true**,
woff2 → **200**, tipografia Geist a video. typecheck 0, prettier ok, 171 test FE verdi, **build di
produzione OK**. Divergenze brand (accent blu, palette light, never-black) lasciate *by design*.
