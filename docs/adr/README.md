Phase 2) F10 Deep Research before engineers code: a single bounded refinement loop
(`for iteration in range(1, max_iter+1)` — structural I7 cap) in `ops/deep_research.py` that generates
sub-queries → searches **SearXNG only** (`ops/searxng.py`, `SEARXNG_URL`, static-guard bans
tavily/ddg/google/serpapi — I9) at hardcoded `concurrency=3` → fetches+extracts → **assesses

All three inference steps ride the existing backend-neutral `InferenceProvider.chat()` (operation
`"ingest"`) — **no new ABC method, no hardcoded backend** (I6). Bounds (`max_iter` default 3,
`token_budget` 100k, `max_queries_per_iter=5`, `concurrency=3`) are **frozen on the
`deep_research_runs` row at start and never re-read** (AQ-v0.5-4); status defaults pessimistically to
`max_iter_reached` and a `finally` writes `error`, so a run is never left `running`; `total_cost_usd`
logged with the $1 anomaly WARNING (I7). New tables `deep_research_runs` + `deep_research_sources`
(**Alembic 0009** — F10 lands before F9, correcting the tentative numbering; `review_items` becomes
0010). REST: `POST /research/start` (202 + `run_id`), `GET /research/runs` (paginated),
`GET /research/runs/{id}` (detail incl. synthesis + sources) mirroring `/ingest/runs` — the surface F9
wires its Deep-Research action to. D3 adds `deep-research.mmd`. ADR-0025 locks sprint v0.5 (M5 Phase 3)
**F9 (HITL review queue) + F12 (multi-format ingest)** before engineers code. **F9:** a new `review_items`
table (**Alembic 0010** — F10 took 0009) scoped by the existing **`vault_id` String** (no `vaults` table,
AQ-v0.5-6), `page_id`/`deep_research_run_id` nullable FKs; items come from **one** source — the
orchestrator's post-write hook enqueues a `new_page` item per generated page and makes **exactly one**
bounded `InferenceProvider.chat()` call (operation `"ingest"`, I6) to pre-generate 1–3 follow-up queries
(timeout + token_budget → NULL on failure; **fire-and-forget, never fails the ingest** — I7/AC-F9-2); REST
`GET /review/queue` (paginated, capped) + `POST /review/queue/{id}/{approve|skip|deep-research}` (the last
delegates to `POST /research/start` and stores `run_id`). The consistency/contradiction graph scan is
**reserved + bounded** (reads `edges`, never FA2 — I2). **F12:** a sole `ingest/extract.py` with a pure
`extract_text(path) -> str` dispatch (pypdf/python-docx/python-pptx/openpyxl — static guard, **no LLM**,
I6/I9); `POST /ingest/upload` widens its accept-list and, on a binary, writes the original to
`raw/sources/` then **synchronously** extracts a companion `<stem>.extracted.md` (Obsidian-valid
frontmatter, I5) — the **watcher ingests only the companion** (binary extension stays OUT of
`_ALLOWED_EXTENSIONS`), so one `ingest_file`, one hash gate (I1), one wiki page; images/AV are
**placeholder-only** and **`unstructured` is deliberately deferred** to M6 (lean deps, I9). D2 adds
`review_items`; D4 adds the 4 review endpoints + widened upload. ADR-0026 closes sprint v0.5 (M5
Phase 4) locking **F13 cascade deletion** — the highest vault-integrity risk in M5 — before engineers
code: a **single-pass, inference-free, FA2-free** `cascade_delete(page_id)` in `ops/cascade_delete.py`.
References to a deleted page are found by a **3-method match** — (a) exact `links.target_page_id`/
`target_title`, (b) slug-normalised `links.target_title`, (c) a **bounded, last-resort** fulltext
fallback over live `wiki/` — where the primary methods read the **`links` back-reference index** (built
incrementally by `persist_links`) and **NEVER rescan the vault** (I1; the (c) fallback is scoped to the
enumerated live-wiki set and skipped on the links-hit happy path). The **preserve-shared rule** is
exact: delete a page **iff** its frontmatter `sources[]` becomes **empty** after removing the source,
else **preserve + prune** the single `sources[]` entry (shared pages are never deleted; source-overlap
`edges` pages are **WARNED, never blocked** — AC-F13-3). Dead `[[wikilinks]]` are rewritten to plain
text (`[[T|alias]]`→`alias`) in the **body only** of **only** the referencing files via the
`frontmatter.loads`/`dumps` round-trip (never editing inside the `---` fences — **I5**;
`test_obsidian_check.py` 15/15 is the gate). The data plane mirrors `delete_file` (soft-delete
`deleted_at`, ADR-0005; `delete_point()`), regenerates `index.md` from live pages (K3 entry auto-removed),
deletes the originating `raw/sources/` file (**AQ-v0.5-5** — no restart resurrection), and bumps
`data_version` **EXACTLY ONCE** so the **debounced** GraphCache recompute fires (**no inline FA2** — I2).
F13 is **migration-free** (reuses `pages.deleted_at`, `links`, the two `edges` endpoint indexes,
`pages.sources`) — **no D2 change**. A **mandatory dry-run** separates plan from apply: read-only
`plan_cascade_delete()` / `POST /pages/{id}/cascade-delete/preview` (200, mutates nothing) returns
`{will_delete, will_preserve_with_pruned_source, wikilinks_to_rewrite, shared_entity_warnings,
match_methods_used}`; the destructive apply is `DELETE /pages/{id}` (200; 404 on a double-delete).
D3 adds `cascade-delete.mmd`; D4 adds both endpoints.
