# Architecture Decision Records — Index

> Last updated: 2026-06-29 · Sprint v0.4 (M4-HARD)
> All ADRs authored by solution-architect; formatted by tech-writer.
> Status values: Accepted | Superseded | Deprecated

ADRs 0001–0006 were authored in sprint v0.1 to lock the walking-skeleton design before
engineers began coding. ADRs 0007–0011 were authored in sprint v0.2 to lock the F17
InferenceProvider abstraction, capability-aware routing, the bounded ingest loop, the
provider_config/ingest_runs/links schema, the MCP transport, and the ingest contract
schemas before engineers began coding. ADRs 0012–0015 were authored in sprint v0.3 to
lock the F4 knowledge-graph engine (4-signal edge weighting), server-side FA2 layout +
coordinate persistence (I2), the dataVersion-debounced GraphCache + GET /graph contract,
and the no-client-layout sigma.js viewer contract before engineers began coding. ADR-0016
was authored at the v0.3→v0.4 transition (M4-GUX Phase 0) to fix the same-type clique
hairball defect: structural edges only (direct link OR shared source), AA/same-type as
weight modulators, sqrt(structural_degree) node sizing, and per-edge `kind` field. ADR-0017
was authored in sprint v0.4 (M4 Phase 1) to lock the F1 3-panel shell: left navigation tree /
center tabbed main (graph now, chat-stub later) / right metadata+relationship preview,
react-resizable-panels for keyboard-resizable panels, a shared single selection key in the
graphStore UI slice, and TanStack Virtual tree virtualization — before engineers began coding.
ADR-0018 was authored in sprint v0.4 (M4 Phase 2) to lock the navigation-rail information
architecture (a persistent ~48px left icon rail switching Pages/Graph/Ingest/Settings sections,
Chat reserved for Phase 3), the read-only Ingest Activity View (virtualized run cards from
GET /ingest/runs with 4dp cost per I7 — explicitly NOT the F9 review queue), the Header Provider
Selector (F17-UI, capability-aware, no hardcoded provider per I6), the Settings section
(context-window F14 + IT/EN i18n + localStorage persistence), react-i18next setup, and the
required ingest_runs migration (status/pages_created/error_message) — visually inspired by
nashsu/llm_wiki but on Synapse's existing dark GitHub palette — before engineers began coding.
ADR-0019 was authored in sprint v0.4 (M4 Phase 3) to lock the F6/F7/F8 chat: NDJSON-over-POST
streaming transport (`POST /chat/stream`, not SSE/WebSocket) with a discriminated token/think/done/error
event schema; capability-aware provider routing via the existing `resolve_provider_config("chat")`
(no hardcoded backend, I6) bounded by token_budget + F16 timeout with per-turn cost logged (I7); a
deliberately minimal Phase-3 retrieval (purpose.md + overview.md light system context — F5 4-phase RAG
and `[n]` citations explicitly deferred to M5); Postgres `conversations`+`messages` persistence
(Alembic 0007, per-message token/cost columns — no chat_runs table, chat never writes ingest_runs);
server-side streaming-safe `<think>` span splitting with raw content persisted un-mutated (F7); and the
I3-critical parse-at-stream-END rule (raw string buffer + parse-once `marked`+`latexToUnicode`+DOMPurify
in a separate chatStore so streaming never re-renders graph/tree/messages — G3 gate) — before engineers
began coding. ADR-0020 was authored in sprint v0.4 (M4-EXT, a parallel PM addition) to lock two new
ingest on-ramps — Feature U (UI document upload, `POST /ingest/upload`, multipart, strict basename-only
path sanitization, text/markdown-only with an F12/M5-naming 415, a byte cap, synchronous `ingest_file`)
and Feature S (scheduled folder import: a new `import_schedules` table via Alembic 0008, a single bounded
asyncio lifespan task rather than APScheduler, hash-compare incremental scan that copies into
`raw/sources/` and lets the watcher ingest, and the hard mounted-path constraint that `source_dir` is a
container path) — both reusing the existing `ingest_file` seam (ADR-0003), the watcher, and the
`ingest_runs` ledger (no new ingest engine, no new run table), before engineers began coding. ADR-0021
was authored in sprint v0.4 (M4-HARD hardening increment) AFTER human browser testing (EC-M4-HCP) to
ratify the post-feedback frontend changes: the **labeled NavRail standard** (72px rail, persistent
10px labels beside icons — supersedes ADR-0018's 48px icon-only rail) and the **M5-item removal**
pattern (Search/Lint/Review/Deep-Search removed from the M4 rail while their `Section` union members
and `SectionRouter` branches are retained for M5 — "features appear when they work"); plus the
**provider-config CRUD contract** for the now read-write LLM Models settings section (ADD via
`POST /provider/config`, DELETE via the existing `DELETE /provider/config/{id}`, no hardcoded model
IDs under user input, server-resolved routing, last-provider deletion permitted with a non-blocking
warning). Approved-with-conditions: wire `conversationHistoryLength` into the chat assembler (I7),
client-require `model_id` before POST (backend field is required), verify the last-provider warning.
They are referenced throughout the codebase as `ADR-XXXX`.
