# Synapse v0.1 — Data flows end-to-end

_Released: 2026-06-28_

Milestone M1: the walking skeleton is wired end-to-end. A file change in the vault triggers the watchdog, updates only the affected Postgres and Qdrant records (never a full rescan), and is immediately queryable via the FastAPI REST layer. This is the foundation on which all subsequent sprints build.

## Added

- **Incremental watchdog indexer (I1)**: file changes update only the affected records in Postgres and Qdrant — no full-rescan, ever (`watcher.py` + watchdog).
- **Postgres + Qdrant data plane**: SQLAlchemy 2 models, Alembic migrations, asyncpg for async queries.
- **bge-m3 embeddings via Ollama (I9)**: reuses the existing TrueNAS Ollama service; no additional infrastructure required.
- **FastAPI REST skeleton**: initial endpoints wiring the watcher output to the data plane.
- **Documentation D1, D2, D4**: C4 architecture diagrams in Mermaid (`docs/architecture/`), ER diagram generated from SQLAlchemy models via `make er` (`docs/er/schema.mmd`), OpenAPI spec auto-generated from FastAPI (`docs/api/openapi.json`).
