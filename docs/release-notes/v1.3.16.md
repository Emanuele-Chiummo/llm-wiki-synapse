# Synapse v1.3.16 — Marker Font Directory Permission Fix

_Released: 2026-07-09_

Single-fix patch: the Marker image now pre-creates and `chown`s the fonts directory to the non-root `marker` user at build time, resolving the `PermissionError [Errno 13]` that surfaced after the GPU fix in v1.3.15 let conversion actually start.

## Fixed

- **Marker font-dir permission** — `marker.util.download_font()` writes to `<site-packages>/static/fonts` on first run but cannot write there as a non-root user, producing `PermissionError [Errno 13]`. The Marker Dockerfile now pre-creates and `chown`s that path to the `marker` user (UID 1000) at build time. The error was masked in earlier releases because the GPU mismatch (fixed in v1.3.15) prevented conversion from reaching the font-download step [F12].
