# Synapse v1.7.1 — post-1.7.0 fixes: output language, knowledge graph, onboarding UX

_Released: 2026-07-15_

Patch release fixing three issues surfaced right after 1.7.0: multi-vault graph collapse, per-vault output language on overview.md, and first-run setup wizard persistence.

## Added

- **Settings → Appearance → "AI Output Language"** — change a vault's generation language after creation (previously only settable in the new-project wizard); saves immediately via `PUT /vault/meta/output-language`. Distinct from the interface language.
- **D5 Home dashboard screenshot** + its Playwright capture spec.

## Fixed

- **Knowledge graph collapsed on multi-vault deployments** — wikilink resolution built its `slug→page` map over all vaults' pages with no `vault_id` filter, so shared slugs resolved cross-vault and produced no graph edge. Measured on an 8-vault DB: 50 graph-eligible links became 5 edges. `_build_resolver_maps` and the graph engine's link query are now vault-scoped. **Single-vault deployments were unaffected.**
- **`overview.md` ignored the per-vault output language** — regeneration resolved language from content detection; an Italian vault built from English sources got an English overview. It now honors `vault_state.output_language` (explicit override → per-vault → this run's analysis → detection).
- **First-run setup wizard reappeared on every reload** — skipping it (defer) only hid it for the current session. Both "completed" and "deferred" now suppress the auto-show across sessions.

## Upgrade notes

- **No migration required.** The graph fix takes effect for new ingests; a single-vault graph is already correct. To reconnect an existing multi-vault graph, re-ingest the affected vault so links re-resolve to the right pages.
