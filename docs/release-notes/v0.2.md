# Synapse v0.2 — Agentic loop + 3 providers

_Released: 2026-06-28_

Milestone M2: the ingest loop is closed and the inference provider layer (F17 — the defining feature) is live with all three backends. Capability-aware routing delegates the full ingest to the CLI agent or runs the bounded orchestrated loop for Local and API providers. All loops are capped by `max_iter` and `token_budget` (I7).

## Added

- **`InferenceProvider` ABC (F17)** with three backends: `OllamaProvider` (Local), `ApiProvider` (Anthropic / OpenAI-compatible), `CliAgentProvider` (claude-agent-sdk).
- **Capability-aware routing**: `supports_agentic_loop=True` (CLI) delegates the entire ingest to the agent; otherwise the orchestrated loop runs — analyze → generate → validate → augment & retry, bounded by `max_iter` + `token_budget`.
- **Bounded loops (I7)**: every loop capped by `max_iter` and `token_budget`; `total_cost_usd` logged per run.
- **MCP standalone server** (FastMCP): initial toolset for external clients (`backend/app/mcp/server.py`).
- **Documentation D3, D7**: sequence diagrams for the ingest loop and provider routing (`docs/sequences/`); first Architecture Decision Records (`docs/adr/`).
