# Synapse v2.1.6 — "past the edge"

Patch release fixing a live-observed Lint failure behind Cloudflare. No schema migrations.

## Fixed

- **"Run Lint" always failed with a bare `524`**: live evidence — the Lint page showed a red
  `524` banner over an empty "No open findings" panel, while the backend had in fact completed
  the scan. `POST /lint/scan` runs the whole bounded scan before answering, holding the HTTP
  connection open for the entire run. On a real vault the semantic pass takes minutes, and the
  Cloudflare tunnel (ADR-0062) terminates any origin request that stays silent past ~100s with
  a 524. The UI could therefore never observe a successful lint scan in production, no matter
  how well the scan itself ran — the failure was in the transport, not in the lint logic. Added
  `POST /lint/scan/start`, which pre-inserts the `lint_runs` row, returns `202 {run_id}`
  immediately and runs the scan in a background asyncio task (the same shape already used by
  `/ops/backfill-domains`, strong task reference included), single-flight per vault (409 while
  one is in flight). The UI now starts the scan there and polls `GET /lint/runs/{id}` until the
  run leaves `running`, then loads its findings. The synchronous endpoint is unchanged for CLI,
  scheduler and test callers. Bounds are untouched: `max_iter` and `token_budget` are still
  frozen on the run row (I7) — the router freezes them itself so the 202 reports exactly what
  the row carries.
- **The "Semantic (LLM)" checkbox had no effect**: `semantic` was declared only as a request
  **body** field while the frontend has always sent it as a **query param**, so the flag was
  silently dropped and every scan ran — and billed — the LLM pass even when unchecked. That
  also made the 524 above far more likely, since the deterministic-only scan would have
  returned in seconds. Both scan endpoints now honour the query param as an override, and the
  client sends the flag in the body as well. The existing L8 test only covered the body shape,
  which is why this shipped; a query-param test now covers the call shape the UI actually
  makes.

## Upgrade notes

No schema migrations. Nothing to reconfigure — the Lint page switches to the background
endpoint on its own. A scan now keeps running server-side even if the browser tab is closed;
its result is visible in the run history and in the open findings list on the next visit.
