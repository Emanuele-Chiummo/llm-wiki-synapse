# Synapse v1.3.3 — PDF-Safe Deep Research and Clickable Citations

_Released: 2026-07-05_

Correctness patch fixing two field-discovered bugs: SearXNG PDF results now go through the Marker/pypdf extractor instead of crashing the run with a Postgres `CharacterNotInRepertoireError`, and chat `[n]` citation clicks now navigate correctly by UUID. Verified with 1,842 backend and 1,761 frontend tests; mypy strict 0 errors; OpenAPI regenerated for the new by-slug endpoint.

## Fixed

- **Deep Research is PDF-proof** — a SearXNG result pointing to a PDF was stored as raw bytes (containing NUL characters), causing Postgres to abort the entire run (`CharacterNotInRepertoireError`). PDFs now go through the ingest extractor (Marker when configured, `pypdf` otherwise), other binaries are skipped with a log entry, all text is NUL-sanitised, and a single unstorable source no longer aborts the run [F10, F12].
- **Chat `[n]` citations open correctly** — clicking a citation passed the derived slug to an endpoint that expects a UUID, returning a 422. Navigation now uses the citation's UUID directly; for historical messages a `GET /pages/by-slug/{slug}` fallback is attempted (using the same slugify logic as retrieval), with a clear toast if the page no longer exists [F5, F6].
