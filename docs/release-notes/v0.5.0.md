# Synapse v0.5.0 — Feature parity core

_Released: 2026-06-30_

Milestone M5: Synapse reaches core feature parity with the Karpathy/llm_wiki pattern. 4-phase retrieval, Deep Research, the HITL review queue, multi-format ingest, cascade deletion, and a remotely-reachable MCP HTTP surface are all live. Backend 829 tests and frontend 609 tests green; ADRs 0022–0033 committed.

## Added

- **4-phase retrieval + `[n]` citations (F5)**: bounded, inference-free `retrieve()` — dense bge-m3 → graph expansion → budget → assembly; `GET /search`, in-chat citations, and save-to-wiki [F5].
- **Deep Research (F10)**: bounded multi-query SearXNG loop — fetch → assess → synthesize → auto-ingest; concurrency 3, cost logged per run (I7).
- **HITL review queue (F9)**: async review items with Create / Deep-Research / Skip actions and pre-generated search queries.
- **Multi-format ingest (F12)**: PDF / DOCX / PPTX / XLSX extraction into Obsidian-valid companion pages.
- **Cascade deletion (F13)**: 3-method slug matching, shared-entity preservation, dead-wikilink cleanup; mandatory dry-run before apply.
- **Remote MCP over HTTP (ADR-0029/0032)**: reachable from claude.ai at `/mcp/server`; runtime enable/disable; read-only by default.
- **UI-settable MCP token (ADR-0033)**: generate/rotate a bearer token from Settings (PBKDF2-hashed, shown once); loopback/Tailscale/LAN allow without token; public Cloudflare sources always require a token.
- **Embeddings toggle (ADR-0030)**: `EMBEDDINGS_ENABLED` flag; graceful degradation to bounded lexical retrieval when off, with no re-scan (I1).
- **OpenAI-compatible embeddings (ADR-0031)**: `EMBEDDING_FORMAT=ollama|openai` plus optional `EMBEDDING_API_KEY`.
- **API base / proxy fix (ADR-0028)**: relative browser base and server-only `BACKEND_PROXY_TARGET` for Tailscale and Cloudflare Tunnel compatibility.
- **MCP server introspection in Settings**: read-only view of the running MCP configuration.
