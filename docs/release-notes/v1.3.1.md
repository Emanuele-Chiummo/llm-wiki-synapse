# Synapse 1.3.1 — pipeline E2E verde + immagine frontend multi-arch 🟢

Patch di release engineering sopra la 1.3.0 «Foundations» — nessuna modifica funzionale all'app.

## Fix
- **Immagine frontend multi-arch in minuti invece che ore**: il bundle Vite (indipendente dall'architettura) ora si costruisce nativamente (`--platform=$BUILDPLATFORM`); solo lo stage nginx resta per-arch. Il build arm64 della 1.3.0 era rimasto oltre 4 ore in emulazione QEMU
- **Job E2E in CI verde per la prima volta**: stack seedato (grafo 340 nodi, provider, conversazioni, sorgenti su disco), bundle costruito con base API e vault corretti, suite Playwright aggiornata alla UI v1.3 — 122+ test contro lo stack live ad ogni push su `main`
- Gate hardware-aware: i test che richiedono Ollama vivo (`E2E_LIVE_CHAT=1`) o GPU reale (`E2E_SOFTWARE_GL`) si auto-skippano dove non hanno senso

## Novità di processo
- Workflow `release-cut` (tag+release da Actions) e `release-notes-sync` (note di release versionate in `docs/release-notes/`)
- Note di release aggiunte retroattivamente a v1.2.1/4/5/6
