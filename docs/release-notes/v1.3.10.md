# Synapse v1.3.10 — Desktop dietro Cloudflare Access & chat in streaming live

> Rende l'app **desktop** (macOS/Windows) funzionante anche da **rete pubblica** dietro
> Cloudflare Access, e porta lo **streaming token-per-token** nella chat col provider CLI.
> **Il backend va ribuildato** (il fix dello streaming è backend). Nessuna migrazione DB.

## 🖥️ Desktop dietro Cloudflare Access

- **Chiamate API via HTTP nativo di Tauri** `[F15][F17]` — l'app impacchetta il frontend
  (origine `tauri://localhost`) e chiama il backend cross-origin; gli header `CF-Access-*`
  facevano scattare un **preflight CORS** che Cloudflare Access respingeva (403), rendendo l'app
  inutilizzabile su hostname pubblico. Ora le richieste passano dal client HTTP **nativo**
  (`tauri-plugin-http`) quando si è nell'app desktop: niente preflight → il **service token CF
  viene accettato**. Il percorso web/PWA resta identico (import dinamico, chunk lazy).

## ✨ Chat

- **Streaming live token-per-token col provider CLI** `[F17]` — il provider CLI
  (claude-agent-sdk) consegnava l'intera risposta in **un solo evento**; ora abilita
  `include_partial_messages` ed emette i `text_delta` incrementali, così la risposta si
  **scrive man mano** anche via abbonamento (nessuna API key). Se la versione dell'SDK non
  supporta i partial, degrada in sicurezza al comportamento precedente (risposta completa a fine).

## ⬆️ Aggiornamento

- **Backend: rebuild necessario** — lo streaming live è lato backend. Ripulla
  `ghcr.io/emanuele-chiummo/llm-wiki-synapse-backend:v1.3.10` (+ frontend). **Nessuna migrazione DB.**
- **App desktop:** nuovo build (macOS/Windows) allegato alla release. Per usarla da **rete
  pubblica** incolla il **service token Cloudflare** in *Impostazioni → Sicurezza*; da Tailscale/LAN
  non serve.
