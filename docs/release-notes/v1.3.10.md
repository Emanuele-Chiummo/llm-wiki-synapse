# Synapse v1.3.10 — Desktop Behind Cloudflare Access & Live Token Streaming

_Released: 2026-07-07_

Makes the Tauri desktop app (macOS/Windows) usable from a public Cloudflare Access-protected hostname by routing API calls through Tauri's native HTTP client instead of the browser fetch (eliminating the CORS preflight that Access rejected). Also adds true token-per-token streaming for the CLI inference provider in chat. Backend rebuild required; no DB migration.

## Added

- **Desktop app works behind Cloudflare Access** — the Tauri app packages a frontend at `tauri://localhost` and calls the backend cross-origin; CF-Access headers triggered a CORS preflight that Access rejected (403), making the desktop app unusable on a public hostname. API calls now go through `tauri-plugin-http` (native HTTP client) when running in the desktop app: no preflight, CF Service Token accepted. The web/PWA path is unchanged (lazy import) [F15][F17].
- **CLI provider streaming — token-per-token**: the CLI (claude-agent-sdk) provider now enables `include_partial_messages` and emits `text_delta` increments, so chat responses stream character-by-character via subscription with no API key required. Degrades gracefully to full-response delivery if the SDK version does not support partial messages [F17].

## Upgrade notes

- Backend rebuild required (the streaming change is backend-side): re-pull `ghcr.io/emanuele-chiummo/llm-wiki-synapse-backend:v1.3.10` and the frontend image. No DB migration.
- To use the desktop app from a public (Cloudflare-protected) URL, paste the CF Service Token in Settings → Security. From Tailscale or LAN no token is needed.
- The iOS IPA (`Synapse-1.3.10-unsigned.ipa`) is functionally unchanged; the update is version alignment only.
