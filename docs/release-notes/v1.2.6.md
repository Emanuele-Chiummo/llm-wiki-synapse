# Synapse v1.2.6 — Bounded Bulk Ingest Concurrency

_Released: 2026-07-04_

Single-fix patch capping bulk ingest concurrency to prevent DB pool exhaustion, GPU overload, and host RAM spikes when many files are dragged into `raw/sources/` simultaneously.

## Fixed

- **Bulk ingest concurrency cap (ADR-0056)** — dragging dozens of files into `raw/sources/` would launch equally many simultaneous ingest tasks, exhausting the DB connection pool, overwhelming the GPU with embedding requests, and spiking host RAM to crash. A semaphore (`INGEST_MAX_CONCURRENCY`, default 3) now processes files in an ordered queue; the _what_ is unchanged, only the _when_ is serialised (I7) [F3].
