# Synapse v1.5.2 ‚Äî Provider config + UX fixes (live-verified)

_Released: 2026-07-11_

Patch: provider-config bugs that broke selecting/using the CLI provider ‚Äî all verified live against real Postgres/asyncpg, not just mocked tests ‚Äî plus UX regressions in the graph, file converter, and review queue.

## Changed

- **More reviews out-of-the-box** ‚Äî `REVIEW_PROPOSE_MIN_PAGES` default lowered `4 ‚Üí 1` (the curated LLM review step now runs on ordinary single-page ingests) and `REVIEW_PROPOSE_MAX_ITEMS` raised `8 ‚Üí 12`; both remain bounded by `token_budget` [F9].

## Fixed

- **Graph node click now opens the page** ‚Äî clicking a node only showed a tooltip; it now selects the node and opens it in NoteView (Obsidian-style) [F4].
- **Overview / page "updated" line showed a raw microsecond ISO** ‚Äî now trimmed to second precision (`‚Ä¶09:44:24Z`), matching the llm_wiki overview footer [F16].
- **File drag-drop into Convert didn't work in the native Tauri (macOS) app** ‚Äî Tauri v2 intercepts OS drag-drop by default; set `dragDropEnabled: false` so the HTML5 drop zone receives files normally (PWA/browser unaffected) [F15].
- **Convert now deletes the source PDF after producing the `.md`** ‚Äî the PDF is removed on success and `sources[]` points at the retained `.md`; best-effort delete never fails the conversion [F12].
- **Review items came out in English on non-English vaults** ‚Äî the propose prompt was not language-aware, the CLI route hardcoded `language="en"`, and the anti-spam gate used title lengths instead of body sizes. Now: mandatory language directive, localized rationales, real body-size gate [F9, F3].
- **`index.md`/`log.md` showed up as bogus automatic groups** ‚Äî meta types (`index`/`log`/`overview`) are now excluded from community labels and top-page previews; they remain graph nodes [F18, F4].
- **`PUT /provider/config/{id}` ‚Üí 500 `MissingGreenlet`** ‚Äî `await session.refresh(row)` now runs before serialization; regression test added for the previously untested PUT endpoint [F17].
- **Ingest resolved the wrong provider** ‚Äî the backend resolver lacked `ORDER BY`, so an older `api` row beat the newer `cli` row; now orders by `created_at DESC` so backend and UI agree on the active provider [F17, I6].
- **Duplicate provider rows piling up** ‚Äî `POST /provider/config` now upserts into a matching non-fallback row instead of always inserting; `ProviderSelector` deduplicates its display (one row per identity, newest = active). Verified live: posting the same provider 3√ó yields one row [F17].
