# Synapse v0.3 — Knowledge graph live

_Released: 2026-06-28_

Milestone M3: the knowledge graph is live with server-side FA2 layout and a sigma.js WebGL viewer. Graph coordinates are pre-computed offline and persisted in Postgres; the client renders them directly — the UI main thread never runs a force-layout (I2), eliminating bottleneck #2 of the original llm_wiki pattern.

## Added

- **4-signal knowledge graph (F4)**: edge weights from direct wikilinks ×3, shared sources ×4, Adamic-Adar ×1.5, and type affinity ×1.
- **Server-side FA2 layout (I2)**: igraph computes coordinates offline; recompute is debounced on `dataVersion` bump and coordinates are persisted in Postgres. sigma.js renders pre-computed coords in WebGL — no client-side force layout, ever.
- **`GET /graph` with cache**: responds with `X-Graph-Cache: HIT` or `MISS`; a cache miss triggers an inline synchronous recompute before responding.
- **React 19 + sigma.js graph viewer**: WebGL rendering with hover-dim and node sizing.
- **Documentation D3, D5**: sequence diagrams updated; Playwright screenshot captures added (`docs/screens/`).

## Fixed

- **Graph edge insert cast**: the edge-insert query used `:signals::jsonb` syntax, which the SQLAlchemy `text()` parser misidentifies (a `:param` followed by `::` is not recognized), causing `recompute()` to fail silently and `GET /graph` to return an empty graph. Fixed by using `CAST(:signals AS jsonb)` instead.
