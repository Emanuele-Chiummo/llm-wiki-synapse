# Synapse v1.5.1 — CLI provider activation fix

_Released: 2026-07-11_

Patch: activating the Claude Code CLI provider (and any catalog vendor) from Settings failed silently because the vendor-catalog tag could not be persisted. Also fixes an unreadable 422 error toast.

## Fixed

- **CLI/vendor provider activation 422** — `POST /provider/config` only accepted `{ingest, chat, lint}` as `operation` values and rejected vendor-catalog ids, so the row was never created and model chips / Test buttons stayed inert. The validator now also accepts vendor-catalog ids [F17].
- **`422 [object Object]` toast** — FastAPI's 422 `detail` is an array of `{loc, msg, type}` objects; the client interpolated it as `[object Object]`. It now renders as readable `field: message` text.
- **Pre-activation provider Test** — "Test connessione"/"Test funzione" on a not-yet-activated vendor now includes the vendor's default model in the inline probe, so it no longer 422s [F17].
- **0-preset vendors (`codex-cli`, `atlas-cloud`)** — activating a vendor with no preset models used to POST a null `model_id` (422); the toggle now reveals the Custom-model input, and choosing/typing a model creates the row [F17].
- **macOS menu-bar (tray) icon** — now the white brand mark (`tray-white.png`) on a transparent background, rendered as-is so it appears white on the menu bar instead of near-invisible dark ink [F15].
