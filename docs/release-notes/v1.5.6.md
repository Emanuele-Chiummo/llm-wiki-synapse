# Synapse v1.5.6 — write toggle, Marker auto-split, UI audit follow-ups

_Released: 2026-07-13_

Bundles two new features (runtime remote-MCP write toggle, Marker `--auto` chapter split) with usability and accessibility fixes from a UI audit of the live instance.

## Added

- **Remote MCP write tools are now toggleable from Settings** (ADR-0072) — *Settings → API & MCP* gains a switch that enables/disables the HTTP MCP write tools (`write_page`, `resolve_review`, `trigger_source_rescan`) at runtime without a backend restart. Persisted in `vault_state.remote_mcp_write_enabled` (migration 0030); exposed via `PUT /mcp/remote-write`; reflected in `GET /mcp/info`. `MCP_REMOTE_WRITE_ENABLED` remains the bootstrap default for fresh vaults [F17].
- **`--auto` mode for the ServiceNow Marker connector** (`tools/marker-converter`): derives module/feature codes from the PDF bookmark outline, splits every module in the book, and defaults to one file per L2 chapter/group — making large multi-module exports (e.g. the 5000-page ITOM book) drop-and-forget in `--watch-dir` mode.

## Changed

- **Localization gaps closed** — user-facing strings that bypassed i18n (`Loading graph…`, `Connections`, `Quick Start`, `Loading`, backend status values `pending`/`cancelled by user`) are now routed through the i18n system with IT + EN translations; `en.json`/`it.json` remain in structural parity.
- **Search has reassuring loading/empty states** — bare "Loading…" replaced by a result skeleton; a "taking longer than expected" message with Cancel/Retry appears after ~4 s; failures use `ErrorState`; empty results show a helpful no-results state.
- **Legibility quick-wins** — sub-12px shared text classes (`.syn-chip`, `.syn-empty-state__eyebrow`, `.syn-meta-row`) raised to a 12 px floor; muted-text tokens rechecked against WCAG AA.

## Fixed

- **Projects page no longer 404s the API in production** — the nginx reverse-proxy regex listed every API prefix except `projects`, so `GET /projects` fell through to `try_files … /index.html` and the SPA received HTML instead of JSON. Added `projects` to the proxied prefixes.
- **Raw technical errors are no longer shown to users** — a new reusable `ErrorState` component (friendly title + Retry + collapsible "Technical details" with a copy button) replaces bare exception text on the Projects page, AI & Models settings, and Search.
