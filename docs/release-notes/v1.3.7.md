# Synapse v1.3.7 — Hotfix proxy `/vault` (produzione)

> Hotfix su v1.3.6. Nessuna nuova funzionalità, nessuna migrazione DB.

## 🐞 Fix

- **`/vault/meta` ora instradato in produzione** `[K1, I5]`. Il reverse-proxy nginx del
  frontend (`frontend/nginx.conf.template`) non includeva il prefisso `/vault` nella lista
  dei path API inoltrati al backend — quindi nell'immagine frontend v1.3.6 la sezione
  **Vault / Meta** (visualizzatore read-only di `schema.md` / `purpose.md`, WS-D8) non
  funzionava: la richiesta cadeva sul fallback SPA e il client, ricevendo HTML al posto di
  JSON, nascondeva la sezione. Aggiunto `vault` al regex nginx (allineato ad `API_PREFIXES`
  in `vite.config.ts`, corretto in v1.3.6 solo lato sviluppo).

## ⬆️ Aggiornamento

Aggiornare direttamente a **v1.3.7** (salta la v1.3.6): ri-pull di **entrambe** le immagini
`ghcr.io/emanuele-chiummo/llm-wiki-synapse-{backend,frontend}:latest` e **ricreare** i
container (un semplice `docker pull` non riavvia). Marker invariato. Nessuna migrazione DB.
