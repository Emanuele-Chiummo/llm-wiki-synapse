# Synapse v1.3.7 — Hotfix: Nginx Vault Proxy in Production

_Released: 2026-07-06_

Hotfix for v1.3.6: the nginx reverse-proxy in the frontend container was missing `vault` in its API-forwarding prefix list, causing the Vault / Meta tree section (added in 1.3.6) to silently return HTML instead of JSON in the shipped image and remain hidden. No new features, no DB migration. Users on v1.3.6 should upgrade directly to v1.3.7.

## Fixed

- **`/vault/meta` not routed in production** — `nginx.conf.template` was missing `vault` in the API-forwarding regex; `GET /vault/meta` fell through to the SPA fallback, returning `<!doctype html>` instead of JSON. The Vault / Meta tree section was hidden in the v1.3.6 container image as a result. Added `vault` to the nginx regex, aligned to `API_PREFIXES` in `vite.config.ts` (which had been corrected in v1.3.6 for the dev server only) [K1][I5].

## Upgrade notes

- Upgrade directly to v1.3.7, skipping v1.3.6: re-pull both `ghcr.io/emanuele-chiummo/llm-wiki-synapse-backend:latest` and `…-frontend:latest` and recreate the containers (a bare `docker pull` does not restart them). Marker image unchanged. No DB migration.
