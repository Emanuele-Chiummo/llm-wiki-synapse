# Synapse v1.1.0 — Convert & Configure

_Released: 2026-07-03_

This release brings PDF-to-markdown conversion via the Marker microservice and in-app provider/model configuration to Synapse, sharply reducing the need to edit environment variables for daily use. Chrome web clipper v1.1.0 ships alongside. Verified with 1,594 backend and 1,501 frontend tests; ruff/black/mypy strict; ER/OpenAPI zero-drift; invariants I1–I9 confirmed.

## Added

- **PDF conversion panel** — drag or select up to 10 PDFs in the new "Convert" section; Synapse converts them via the Marker microservice and auto-ingests the result. Per-file progress (pending → converting → done/error) and a "Marker offline" badge when the service is unreachable. Marker remains optional; the default `pypdf` extractor needs no extra service [F12].
- **In-app setup wizard** — first-run flow verifies backend connectivity → provider/model → PDF extractor → done; skippable and re-openable from "Get Started" [F16].
- **Eight settings out of environment variables**: PDF extractor + Marker URL/timeout, monthly cost alert, vector embeddings on/off + format, overview language, wikilink enrichment. Each field shows a Default/Custom badge and a "Restore default" button. Settings panel reorganized from 14 sections into 5 plain-language groups: Get Started · AI & Models · Sources & PDF · Output & Appearance · Advanced [F16].
- **`app_config` table** — key/value overrides with `env → override` precedence; reset via `DELETE /config/app/{key}`. Protected by the same auth token as the rest of the API (ADR-0053) [F16].
- **Chrome web clipper v1.1.0** [F11].

## Changed

- **Logo shown once** — the Synapse mark no longer appears twice in the sidebar; it remains in the header [F1].

## Fixed

- **Virtualized lists no longer start at 0 px height** — page tree and message history displayed a blank screen until the first resize; height is now correct on mount [I4].
- **`renderMarkdown` protected on empty/null input** — no more console noise on the empty preview pane [F1].
