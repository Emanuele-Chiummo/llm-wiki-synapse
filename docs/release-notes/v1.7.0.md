# Synapse v1.7.0 — llm_wiki 1:1 core parity, link-regression fix, editorial redesign

_Released: 2026-07-14_

Aligns the three core operations (Ingest · Review · Lint) and the generated page types 1:1 with nashsu/llm_wiki v0.6.3, fixes the 1.6.0 wikilink-density regression, brings new-vault onboarding to parity, and refreshes the frontend with an editorial design language. Backend suite 2,933 passed, 4 skipped; live-tested end-to-end on the real stack (Postgres 16 + Qdrant + Claude CLI/Haiku), achieving 12/12 parity bands. Decisions: ADR-0076..0083.

## Added

- **Block-based ingest pipeline** (ADR-0076) — `ingest_pipeline_format` flips to `"blocks"` (default); `"json"` kept as a rollback lever (removal slated for 1.8). In blocks mode every provider — Local, API, and CLI — runs the block loop via `provider.complete()`; the delegated agent loop is used only in `"json"` mode [I6].
- **Schema-driven page routing + open page-type set** (ADR-0077) — `wiki/schema.py` parses the `schema.md` "Page Types" table into an authoritative `type → dir` map; custom types (thesis, methodology, character, …) persist as a raw `page_type` string.
- **New-vault onboarding wizard** (ADR-0081; migration 0032) — a 3-step modal (name + parent dir → mandatory AI output language → scenario template) that auto-activates the new vault. Five scenario templates carry `extra_dirs` + custom page types; per-vault `vault_state.output_language` drives the ingest language directive. `GET`/`PUT /vault/meta/output-language`.
- **`index.md` "Recently Updated"** — a code-owned bounded catalogue section alongside the K3 per-type catalogue (ADR-0078); manual `POST /ops/overview/regenerate`.
- **Parity E2E harness** (ADR-0083) — deterministic 3-doc corpus + `scripts/parity_e2e/compare.py` with tolerance bands and a total-links ≥ 1.5.6 baseline regression sentinel.
- **Editorial frontend redesign** — new design-token language (ink `#0F1729`, cool neutrals, categorical page-type palette), grouped nav rail (Create / Understand / Maintain), editorial Wiki reader with a Connections panel (server FA2 coords, I2), Review decision-trace card. Light + dark, EN/IT parity.
- **Home dashboard second pass** — composition hero (total pages over a jewel-tone per-type bar + legend), semantic KPI zero-states (lint `0` → green "clean"), primary ingest action, color-coded review-type chips, graph legend hides zero-count node types.

## Changed

- **Wikilink density restored at the prompt** — prompts carry prominent, repeated `[[slug-form-wikilink]]` guidance and an analysis "Connections to Existing Wiki" section. Link resolution ~90–100 % on the 1:1 E2E. Wikilinks are emitted as bare kebab-case slugs (`[[cloud-cost-explorer-api]]`, with the `[[slug|Display]]` escape hatch). Enrich post-pass defaults **off** (ADR-0076).
- **Review** (ADR-0079) — auto-resolve sweep fires on queue drain (not per run); Create Page defaults to a deterministic stub (`mode="stub"`), full-LLM generation is explicit (`mode="generate"`); Dismiss now confirms first; block-loop REVIEW blocks are enqueued.
- **log.md** entries use llm_wiki's `## [YYYY-MM-DD] ingest | Title` format; **overview.md** regenerates once per queue-drain batch — a whole-wiki synthesis reading `purpose.md` + the existing-page digest — rather than per source.
- **Generation covers all derived page types** — the prompt asks for query, comparison, and synthesis pages from open questions / commensurable subjects / cross-cutting conclusions, and one page per distinct named entity or concept.
- **Review/overview single-call seams route through `complete()`** (single-turn) with timeouts raised to 120 s for CLI subprocess cold-start; degrade-safe.
- **Lint** (ADR-0080) — assessed at parity (0.74 broken-link threshold, deterministic fixes, `proposal_origin="lint"` review routing); bounded semantic loop and on-demand Send-to-Review are deliberate supersets.
- **Orchestrator decomposed** — 3,851 → 1,493 lines behind a re-export facade.

## Fixed

- **The 1.6.0 wikilink-density regression** — diagnosed as prompt-only; the 6-type JSON scaffold had buried the single `[[wikilink]]` instruction. The ported prompts restore it prominently.
- **Dangling wikilinks under the CLI provider** — the delegated agent linked to entities it never materialised (169 links, 1 graph edge); routing CLI through the block loop lifts link resolution to ~90–100 %.
- **Block-loop non-convergence** — the validator was counting app-managed `index/log/overview` FILE blocks as routing errors, forcing retries to `max_iter`; those blocks are now skipped.
- **`overview.md` stayed an empty stub** — regeneration used `chat()` (agentic CLI loop) and timed out; the drain-time `complete()` path now produces a rich synthesis.
- **Review auto-resolve did nothing under the CLI** — the sweep/propose seams timed out via `chat()`; with `complete()` + 120 s ceiling the sweep resolves items again.
- **Frontend showed the wrong vault** — the dashboard never synced `vault_id` from `/status`; the status poll now propagates the active `vault_id`.
- **Onboarding no longer strands a new vault un-activated** — the wizard activates + reloads.

## Upgrade notes

- **Run Alembic migration `0032`** — additive, nullable `vault_state.output_language`; NULL = auto.
- **`ingest_pipeline_format` now defaults `"blocks"`** — set `"json"` to roll back to the 1.6.x loop (removal slated for 1.8).
- **`wikilink_enrich_enabled` defaults false** — set true to restore the post-pass.
- **Review/overview single-call timeouts default 120 s** (`review_sweep_timeout_seconds`, `review_propose_timeout_seconds`, `overview_timeout_seconds`); lower for API/Ollama-only deployments if desired.
