# Synapse v1.5.4 — llm_wiki 1:1 parity — ingest boundaries, fuzzy lint, review dedup

_Released: 2026-07-12_

Patch: closes five function-by-function divergences found auditing Synapse against nashsu/llm_wiki across ingest, review, and lint. All prompt and logic changes are provider-neutral (I6) and covered by unit and integration tests.

## Added

- **Subject-boundary rules in the ingest prompts** — the analyze, generate, and re-ingest merge prompts now instruct the model to keep every claim attached to its exact subject and never transfer them between subjects sharing keywords. Direct port of nashsu/llm_wiki (ingest.ts:1949/2070-2072/2792-2793); prevents claim-bleed. Applied to all three backends via the shared `GENERATION_SCAFFOLD` [F3].
- **Delegated (CLI) route source-summary guarantee** — the delegated ingest route now runs the same deterministic "ensure exactly one source page" fallback as the orchestrated route, as an additive post-run step. Mirrors llm_wiki's `hasSourceSummary` fallback [F3].
- **Mandatory output-language directive on the delegated route** — the CLI agent now receives an explicit "write page bodies and frontmatter `lang` in the vault language" instruction, so delegated pages no longer silently drift to English [F3].

## Changed

- **Review auto-resolve sweep exits early** — the Pass-2 LLM sweep now stops after the first empty batch (llm_wiki sweep-reviews.ts:307-310 parity), cutting unnecessary provider calls (I7) [F9].

## Fixed

- **Broken-wikilink suggestions now re-point instead of spawning stub pages** — added a typo-tolerant fuzzy fallback (Levenshtein, threshold 0.74) — a verbatim port of llm_wiki `suggestBrokenTarget`. Suggestion-only: it never creates a graph edge [K2].
- **Review queue no longer bloats on re-ingest** — `confirm` items now dedup on (type + normalized title) like every other review type (llm_wiki `reviewIdFor` parity); resolved confirmations are never re-opened [F9].
