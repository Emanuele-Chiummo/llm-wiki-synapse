## 3. G1–G4 verdicts

### G1 — No full-rescan (I1) — PROVEN-NOW

**Evidence:**
- `engine.py:_load_data()` — SQL queries only on `pages` + `links` tables; zero `os.walk`, `pathlib.glob`, or vault filesystem access (code inspection confirmed).
- `engine.py:_persist_results()` — targeted `UPDATE pages SET x=..., y=...` per node; `DELETE FROM edges WHERE vault_id=... + bulk INSERT edges`. No full table truncate.
- T-INC-GRAPH-001: 1 new file → row count 2→3 (not full delete+recreate).
- T-INC-GRAPH-002: pre-existing page rows survive new ingest.
- T-INC-017 (test_ingest_incremental.py): watcher.py has no `os.walk`/`.rglob()` calls.

**Verdict: PROVEN-NOW (no live infra needed)**

### G2 — Graph layout cached, no main-thread freeze (I2) — PROVEN-NOW (static) + DEFERRED-TO-LIVE (runtime perf)

**Proven-now (static/unit evidence):**
- 13 GraphCache tests (T-GCACHE-001..013): debounce collapses bursts to 1 recompute; HIT on same data_version skips FA2; injectable clock proves no real sleep in tests.
- 13 GET /graph API tests (T-GRAPI-001..013): X-Graph-Cache: hit on second call; X-Graph-Cache: miss on stale version; full schema on HTTP 200.
- 22 no-client-layout tests (T-NCL-001..022): bundle scan + source import scan confirm graphology-layout-forceatlas2, d3-force, ForceAtlas2, fa2Worker, and 6 other forbidden patterns are ABSENT from both the built bundle and all non-test source files.
- GraphViewer.tsx: sigma instantiated with `new Sigma(graph, container, {...})` using pre-computed server x/y; no layout algorithm called client-side (code inspection).
- graphStore.ts: all subscriptions use typed selectors + `useShallow`; no whole-store subscription (I3 pre-compliance also confirmed).

**Deferred-to-live (runtime):**
- T-E2E-G2-001: Playwright PerformanceObserver asserting no longtask >50ms. Run: `cd frontend && npx playwright test e2e/graph-perf.spec.ts --config playwright.config.ts`
- T-E2E-G2-002: X-Graph-Cache: hit assertion via Playwright page.request.get(). Same run command.

**Verdict: PROVEN-NOW for static/architectural proof; DEFERRED-TO-LIVE for browser runtime measurement**

### G3 — Streaming / no per-token heavy work (I3) — N/A UNTIL v0.4

G3 applies to the chat streaming path, which is not implemented in v0.3 (v0.4 deliverable). Pre-compliance confirmed:
- graphStore.ts uses typed selectors everywhere; `useGraphStatus()` and `useGraphMeta()` use `useShallow()`.
- No component calls `useGraphStore()` without a selector parameter.

**Verdict: N/A (v0.4) — pre-compliance CONFIRMED by code inspection**

### G4 — WebGL ≥60fps, bounded DOM (I4) — DEFERRED-TO-LIVE

**Static evidence (pre-conditions confirmed):**
- GraphViewer.tsx: single sigma.js canvas element; no per-node DOM elements created.
- DOM structure comment in GraphViewer.tsx: `<div#graph-root> + <div#sigma-container> + <canvas> + overlay divs ≈ 6 total DOM elements` (not 200+).
- sigma.js uses WebGL (Canvas API with WebGL context), not SVG or Canvas2D+JS layout.

**Deferred-to-live:**
