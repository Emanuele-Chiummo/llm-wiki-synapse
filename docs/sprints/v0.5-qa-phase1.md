## 5. Performance Invariant Gates (Phase 1-relevant)

### G3 / I3 — No per-token markdown parse; citation decoration single-pass (ADR-0022 Do-NOT #6)

**Code review:** `decorateCitations.ts` has exactly one import (`CitationRef` type from
chatStore). It imports no markdown parser; its module-level interface imports confirm zero
`renderMarkdown` dependency. It is a single-pass regex replacement over the already-parsed
HTML string, memoized on `(rawHtml, citationKey)`.

**Test coverage:**
- `chat-parse-once.test.ts::G3 — parse-once invariant::streaming 100 tokens: renderMarkdown
  is NOT called during token phase` — asserts `spy` call count == 0 during 100 `appendToken`
  calls.
- `chat-parse-once.test.ts::after finalizeTurn: parse selector fires exactly once (AC-G3-3)`
  — asserts the messages-length subscriber fires exactly 1 time on `finalizeTurn`.
- `ChatMessage.test.tsx::decorateCitations — memoization: same inputs return the same string
  reference` — confirms zero re-processing on re-render.
- `ChatMessage.test.tsx::streaming 100 tokens: citations never mutated during stream (I3)` —
  asserts messages array not touched during 100 streaming tokens.

**Verdict: GATE PASS.** The G3 invariant is verified: renderMarkdown is called 0 times
during streaming and decorateCitations adds no second markdown parse.

### I1 / AC-F5-5 — retrieve() and GET /search are read-only; data_version unchanged

**Test coverage:**
- `test_retrieval.py::test_ac_f5_5_data_version_unchanged` — reads data_version before
  and after `retrieve()` with SQLite, asserts `before == after == 7`.
- `test_api.py::TestGetSearch::test_search_does_not_bump_data_version` — ASGI client test
  asserts `/status` data_version is identical before and after `GET /search`.
- `test_api.py::TestGetSearch::test_search_data_version_in_response` — asserts
  `search.data_version == status.data_version` (snapshot taken before assembly, AC-F5-5).

**Verdict: GATE PASS.**

### I7 — CliAgentProvider.chat() bounded by CHAT_AGENT_MAX_TURNS + no runaway

**Test coverage:**
- `test_cli_chat.py::test_chat_bounded_by_chat_agent_max_turns_env` — sets env to 3,
  asserts SDK receives `max_turns=3`.
- `test_cli_chat.py::test_chat_default_max_turns_is_eight` — unset env, asserts `max_turns=8`.
- `test_cli_chat.py::test_chat_invalid_max_turns_falls_back_to_default` — sets env to "0"
  (non-positive), asserts fallback to `max_turns=8`.
- `test_cli_chat.py::test_chat_no_api_key_raises_clean_config_error_not_fake_stream` —
  asserts `ValueError` raised before generator returned (Do-NOT #9 verified; no fake stream).

**CHAT_AGENT_MAX_TURNS default:** 8 (confirmed in `cli.py::_DEFAULT_CHAT_AGENT_MAX_TURNS`).
**Invalid value fallback:** non-positive integer → default, with WARNING logged.
**NB-4 cost:** float 0.0 (not Decimal — see notes §1).

**Verdict: GATE PASS.**

### I6 — provider.chat() signature unchanged across all 3 providers

**Test coverage:**

**Mock contract documentation (for CI replacement):**
- OllamaProvider / ApiProvider: `_MockProvider` in `test_chat.py` and `_RecordingProvider`
  in `test_chat_endpoint.py` — both accept `(messages: list, retrieval_context: str)` and
  yield str deltas; they record `received_retrieval_context` for assertion.
- CliAgentProvider: `_install_fake_sdk()` in `test_cli_chat.py` replaces
  `claude_agent_sdk` with a `_FakeClient` that accepts `ClaudeAgentOptions` kwargs,
  exposes `query()` + `receive_response()`, and yields `_FakeAssistantMessage` /
  `_FakeResultMessage` objects matching the SDK shape documented in R3.
- To replace with real providers when GPU/API key is available: remove `monkeypatch.setenv`
  for the fake credentials, remove `_install_fake_sdk` call, and let the provider hit the
  live endpoint. The assertion structure is unchanged.

**total_cost_usd tracking:** `test_cli_chat.py::test_chat_records_real_sdk_cost_when_present`
asserts `acc.total_cost_usd == pytest.approx(0.042)` (real SDK cost path).
`test_chat_falls_back_to_zero_cost_with_warning` asserts `acc.total_cost_usd == 0.0` +
