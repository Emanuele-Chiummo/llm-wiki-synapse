## 3. Avvia lo stack (un solo comando) — consigliato per il Mac

Synapse è **client/server**: l'UI (web/PWA/desktop) è un client sottile, il motore è il backend +
Postgres + Qdrant + Ollama. Quindi "aprire l'app" da solo non basta: serve lo stack su.
Sul tuo Mac usa lo **stack completo in un comando** (frontend + backend + postgres + qdrant):

```bash
make dev            # = docker compose -f docker-compose.yml -f docker-compose.dev.yml up
```
Requisito unico: **Ollama** attivo a `http://localhost:11434` (Ollama.app). Il container esegue
da solo `alembic upgrade head` (quindi applica anche 0014/0015/0016).

Poi apri **http://localhost:5173** (UI). API su **http://localhost:8000**.
```bash
curl -s localhost:8000/status | jq          # health check backend
```
Stop: `make dev-down`.

> **Alternativa "produzione TrueNAS"** (backend + postgres in container; Qdrant/Ollama/SearXNG
> esterni; **niente frontend containerizzato** → il frontend lo servi a parte):
> `make up` + `make logs` + `make migrate`, e per l'UI `cd frontend && npm ci && npm run dev`
> (proxy → `:8000`, override con `BACKEND_PROXY_TARGET`).

Verifica migration applicate (deve arrivare a `0016`):
```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml exec synapse-backend alembic current
```

Infine in UI: **Impostazioni → LLM Models** → aggiungi un provider (Local Ollama / API / CLI) —
senza, ingest e chat non partono.

## 4. Usare il tuo abbonamento Claude (provider CLI, costo $0) — ADR-0042

Il provider **CLI** (`claude-agent-sdk`) può girare sul tuo **abbonamento Pro/Max** invece che a
consumo con una API key. Funziona **anche dentro il container**, via un token OAuth a lunga durata.

**Passo 1 — genera il token (una volta, sul Mac loggato con l'account Pro/Max):**
```bash
claude setup-token          # stampa un token OAuth (sk-ant-oat01-...), valido ~1 anno — copialo
```

**Passo 2 — mettilo nel `.env` e NON impostare la API key:**
```
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-...     # il token del passo 1
# ANTHROPIC_API_KEY  ← lascialo NON impostato (nemmeno vuoto)!
```
> ⚠️ **Trappola di precedenza:** se `ANTHROPIC_API_KEY` è impostata (anche `=""`), **vince lei** e
> ti fa pagare a token. Per l'abbonamento deve essere **completamente assente**.

**Passo 3 — riavvia lo stack** (`make dev`): il container legge `CLAUDE_CODE_OAUTH_TOKEN`, il `claude`
CLI incluso nell'SDK autentica via abbonamento. Nell'ingest/chat il `total_cost_usd` sarà **$0.00**.

**Passo 4 — in UI:** **Impostazioni → LLM Models** → aggiungi un provider **CLI** e scegli il modello
(es. `claude-opus-4-8`). Fai un ingest o una chat: costo $0 nell'Activity panel.

I tre modi di autenticazione (ne scegli **uno**), documentati anche in `.env.example`:

| Variabile | Provider | Costo |
|---|---|---|
| `ANTHROPIC_API_KEY` | API / CLI | a token |
| `CLAUDE_CODE_OAUTH_TOKEN` | **CLI (abbonamento, in container)** ✅ | **$0** |
| `CLAUDE_CODE_USE_SUBSCRIPTION=true` | CLI (login host/montato) | $0 |

> Nota Tauri: il guscio desktop resta un client HTTP verso `localhost:8000` — non cambia nulla per
> l'auth. L'abbonamento è deciso dal **backend**, non da Tauri.

## 6. Smoke manuale (EC‑M6‑HCP) — checklist

Spunta ognuna nel browser/app:

**Core (Ingest · Query · Lint)**
- [ ] Crea progetto → **Sources**: importa un documento → parte l'ingest (Activity panel).
- [ ] Compaiono pagine wiki, entità/concetti; **Graph** mostra i nodi/archi.
- [ ] **Chat**: fai una domanda → risposta con citazioni `[n]`.
- [ ] **Lint** (nuovo): sezione Lint → **Run Lint** → compaiono findings (orphan/missing‑xref/contradiction/stale/missing‑page) → prova **Fix** (missing‑xref/missing‑page) e **Acknowledge** (gli altri) e **Dismiss**.

**Web Search (nuovo pannello)**
- [ ] **Impostazioni → Web Search**: imposta l'URL SearXNG (es. `http://host.docker.internal:8888`), categorie, max query → salva; il badge "source" passa a **db**.
- [ ] **Deep Research** su un gap → esegue query SearXNG → sintetizza → auto‑ingest.

**Web Clipper (nuovo pannello + estensione)**
- [ ] **Impostazioni → Web Clipper**: **Enable** ON → **Genera token** (copia il valore mostrato una‑tantum).
- [ ] Carica l'estensione: Chrome → `chrome://extensions` → *Modalità sviluppatore* → *Carica non pacchettizzata* → seleziona la cartella **`extension/`** → copia l'**ID** dell'estensione.
- [ ] In Impostazioni → Web Clipper, aggiungi `chrome-extension://<ID>` in **Allowed origins** → salva.
- [ ] Nell'estensione (Options): base URL `http://localhost:8000`, incolla il **token**.
- [ ] Vai su una pagina web → clicca l'estensione → **Clip** → il markdown finisce in `vault/raw/sources/` e viene **auto‑ingerito** (lo vedi nell'Activity panel).
- [ ] Prova gli errori attesi: token errato → 401; se disabiliti l'ingress → 503.

**Altro**
- [ ] **Review** queue: un ingest genera proposte → Approve / Deep‑Research / Skip.
- [ ] **Cascade delete**: elimina una source → la sua pagina wiki e i wikilink morti spariscono; entità condivise restano.

**PWA**
- [ ] `cd frontend && npm run build && npm run preview` → apri l'URL → in Chrome *Installa app* → verifica shell offline; le chiamate API NON sono cachate (nessun dato stantìo).

**Desktop (Tauri)**
- [ ] `cd frontend && npm run tauri:build` (serve Rust) → installer in `src-tauri/target/release/bundle/…` → lancia l'app.
- [ ] In alternativa: `npm run tauri:dev` per la finestra desktop in sviluppo.

**Matrice 3 provider (I6)**
- [ ] Ripeti un ingest+chat con provider **Local (Ollama)**, **API**, **CLI**.

## 7. Se qualcosa non va — check rapidi

| Sintomo | Verifica |
|---|---|
| `/status` non risponde | `make logs`; Postgres su? `docker compose ps` |
| Ingest non parte | provider configurato in Impostazioni → LLM Models? `EMBEDDING_URL`/`OLLAMA_URL` raggiungibili? |
| Deep Research 503 | SearXNG impostato (Impostazioni → Web Search o `SEARXNG_URL`)? |
| Clip 401 | token corretto nelle Options dell'estensione? Enable ON? |
| Clip 403 | `chrome-extension://<ID>` negli Allowed origins? |
| Migrazioni mancanti | `docker compose exec synapse-backend alembic upgrade head` |
| Frontend non chiama il backend | proxy: backend su `:8000`? `BACKEND_PROXY_TARGET` giusto? |
