## Two portability bugs found in live testing and fixed (commit a06fe7f)

1. **C4 diagrams** had a leading `<!-- Generated -->` HTML comment → broke both the
   `startswith('C4Context')` QA test and Mermaid rendering (C4 must start with the
   diagram keyword; Mermaid comments are `%%`). Removed.
2. **alembic.ini** was at `backend/alembic/alembic.ini` but `script_location = alembic`
   + `prepend_sys_path = .` require it at `backend/` root (Dockerfile `COPY alembic.ini .`
   expected it there). Moved to `backend/alembic.ini`.

These slipped the sandbox gate because neither is verifiable without live services —
both were correctly tracked as deferred-to-live (AC-DC-1) and the human checkpoint caught
them, exactly as intended.

## Process lessons carried into v0.2

- QA must re-run the full suite **after** tech-writer edits, not before (the C4 comment
  was added post-QA-run).

  instead of at manual test.
