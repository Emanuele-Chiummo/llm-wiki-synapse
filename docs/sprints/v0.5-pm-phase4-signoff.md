### FAIL → FIXED → PASS cycle (process note)

The QA gate initially returned FAIL with two P1 defects:

- **DEFECT-F13-001** (I1 gate): `_method_c_fulltext` invoked unconditionally on the links-table-hit happy path, violating ADR-0026 §3.1. Fix: `if not all_found_ids:` guard before the fallback call. T-CD-026 flipped from xfail to PASS.
- **DEFECT-F13-002** (I5 gate): `frontmatter.dumps()` reordered YAML keys alphabetically on every dead-wikilink rewrite and sources[] prune, violating ADR-0026 §4.3 "byte-for-byte" claim and I5 (unnecessary git diffs in Obsidian). Fix: new `_rewrite_body_preserving_frontmatter` helper (raw `---` split, no PyYAML round-trip) for wikilink rewrites; `sort_keys=False` for `_prune_sources`. T-CD-024b flipped from xfail to PASS.

This cycle is a **process win**: the AsyncMock-only backend suite (which passed before QA gate) could not catch either defect because it does not execute real DB queries or real file I/O. The QA-authored real-filesystem tests (T-CD-024b, T-CD-026) caught both. The gate mechanism worked exactly as designed.

After both fixes: 674 backend / 526 frontend, 0 xfail/xpass, static analysis clean on all layers (ruff, black, mypy, tsc, eslint).
