## 1. I2 — server-side layout, cached, NO client force layout (HEADLINE) — **APPROVE**

**Server side (engine.py):**
- Force layout runs ONLY in `GraphEngine.recompute()` — `g_weighted.layout_fruchterman_reingold(weights=..., niter=500)`.
  This is the python-igraph (R9, I9) force-directed layout explicitly sanctioned by ADR-0013 §1
  as the "FA2-family" layout (`layout_fruchterman_reingold` / `layout_drl`). Not a deviation —
  it is exactly what my own ADR-0013 §1 architect note authorised. Library locked to igraph. ✅
- Determinism: `igraph.set_random_number_generator(_SeedableRNG(FA2_SEED))`, `FA2_SEED=42`
  (env `GRAPH_LAYOUT_SEED` override). `TestFA2Determinism.test_same_coords_two_runs` asserts
  byte-stable coords across two runs (|Δ| < 1e-9). Matches ADR-0013 §2. ✅
- Coords persisted to `pages.x/y` via column-level `UPDATE pages SET x,y WHERE id` inside one
  transaction with the edges delete-then-insert (`_persist_results`). Matches ADR-0013 §3/§5. ✅

**Cache (cache.py):**
- `GraphCache` debounces recompute on `data_version` bump (`notify_bump` → `fire_at = clock()+window`,
  default 5s; burst collapse resets the deadline). Injectable clock. Bounded queue: max 1 in-flight +
  1 pending (`_in_flight` / `_pending`), exactly one follow-up after a run. Matches ADR-0014 §2/§3. ✅
- HIT (`_marker == current_version`) returns the in-process `_snapshot` with NO recompute — no FA2. ✅
- MISS runs ONE inline recompute behind the shared in-flight guard (no concurrent FA2). ✅

**Endpoint (main.py GET /graph):**
- Synchronous 200. `X-Graph-Cache: hit|miss` set from the `cached` flag. No 202 path. Matches
  ADR-0014 §5/§6 and AQ-v0.3-3. ✅

**Frontend (GraphViewer.tsx + graphTransform.ts + graphStore.ts):**
- `buildGraphologyGraph` assigns `x: node.x, y: node.y` verbatim from the server response; no
  layout function is called anywhere. Sigma is instantiated over the prebuilt graphology graph and
  renders the fixed positions; no rAF physics loop, no supervisor. ✅
- `package.json` dependencies = graphology, react, react-dom, sigma, zustand. **No**
  forceatlas2 / d3-force / graphology-layout-* dependency. Confirmed absent from `package.json`
  **and** `package-lock.json` (transitive deps included). ✅
- Enforcement is genuine and double-layered:
  - eslint `no-restricted-imports` blocks forceatlas2, d3-force, @antv/layout, graphology-layout
    (+ random/circular variants) — P0 message.
  - `no-client-layout.test.ts` scans BOTH the built `dist/assets/*.js` bundle AND non-test source
    imports for 10 forbidden patterns (forceAtlas2, fa2Worker, forceSimulation, d3-force, etc.),
    failing with a P0 escalation message on any hit. Both layers are real, not stubs. ✅

**Verdict: I2 APPROVE.** No client-side layout exists or can be introduced without tripping two
independent gates. FA2 is server-side, seeded, deterministic, and cached. **No P0 block.**

## 2. Edge-weight formula (ADR-0012) — **APPROVE**

engine.py implements exactly:
`w = 3.0·direct + 4.0·shared + 1.5·aa + 1.0·same_type` (lines computing `w` in `recompute`). ✅
- `direct`: both link orientations counted separately via `pair_direct` (canonical key). ✅
- `shared`: `len(sources_A ∩ sources_B)` over JSONB arrays; NULL→∅. ✅
- `aa`: `Σ_{c∈N(A)∩N(B)} 1/ln(deg(c))`, deg>1 guard, over the undirected unweighted resolved-link
  graph. Candidate-pair restricted (not O(N²)). ✅
- `same_type`: 1 iff both non-NULL and equal; NULL≠NULL. ✅
- Persisted iff `w>0`. Pairs canonicalised (smaller UUID first) via `_canonical_key`. ✅
- `signals` JSONB persists the per-term breakdown for audit. ✅

`test_graph_engine.py` asserts **exact hand-computed** values, not just smoke:
- P3–P4 `== 7.0` (direct 3 + source 4, AA 0, type 0) — exact. ✅
- signals isolation on P3–P4: direct=3.0, source=4.0, aa=0.0, type=0.0 — exact. ✅
- P1–P2 type signal == 1.0 (both entity); NULL-type pair type signal == 0.0 (NULL≠NULL). ✅
- Lower bounds for AA-bearing pairs: P1–P2 ≥ 11, P1–P4 ≥ 8, P2–P4 ≥ 5. ✅
- P3–P5 absent — and the QA fixture correctly applied the AQ-1 correction (P5 type=`person`,
  not `concept`), so weight=0 → not persisted. The architect-flagged fixture reconciliation was
  honoured. ✅

**Verdict: APPROVE — formula is exact and exactly tested.**

## 3. I1 — incremental (no rescan) — **APPROVE**

- `_load_data` issues SQL only against `pages` + `links`. No `os.walk` / `glob` / vault filesystem
  access in engine.py or cache.py. ✅
- `_persist_results` does column-level `UPDATE pages SET x,y` per node + `DELETE FROM edges WHERE
  vault_id` then re-insert; no full pages-table rewrite. ✅
- One ingest → that page's rows + ONE debounced recompute. FA2 repositioning all nodes is the
  documented and CORRECT global-layout semantics (ADR-0013 §4); AC-F4-9 asserts row-level
  incrementality, not coord stability. This is **not** an I1 violation. ✅

**Verdict: APPROVE.**
