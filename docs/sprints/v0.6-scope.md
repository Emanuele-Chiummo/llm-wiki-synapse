## 1. Sprint Objective

Take Synapse from "feature parity core" (M5) to **shippable v1.0**: a self-hosted product a
new user can install and use day-one, at functional and graphical parity with the reference
app `nashsu/llm_wiki`. The headline theme is **closing the loop**: the third Karpathy
operation (Lint) lands; the ingress surface (web clipper) is added **securely**; the app is
**packaged** (PWA + Tauri v2) with a CI/CD that actually runs the test suite; and the full
D1–D7 documentation set is completed and consistent.

**Headline invariants for M6:**
- **I7 (bounded loops)** — the new Lint-fix loop has `max_iter` + `token_budget` + cost log.
- **I6 (pluggable inference)** — Lint's semantic pass routes through `InferenceProvider`; no hardcoded backend.
- **I8 (docs-as-DoD)** — M6 is the final docs gate: every D-artifact complete and consistent, ER matches schema.
- **I9 (do not reinvent)** — no Tavily/alt-search; SearXNG only; no local embedding reimplementation.

**Parity-reference rule:** the `llm_wiki` audit is advisory input, not a spec to copy.
Where the reference app made a security/architecture mistake, Synapse must NOT repeat it.
The audit's standout finding — the reference's clip server on `:19827` is unauthenticated,
accepts drive-by cross-origin writes, and writes to an attacker-controlled base path — is an
explicit **anti-pattern**: Synapse's F11 ingress MUST be authenticated, origin-checked,
body-capped, and path-safe (see F11 ACs).

## 3. Per-feature Acceptance Criteria

### K2 — Lint-fix loop
- **AC-K2-1** `ops/lint.py` runs a bounded loop (`LINT_MAX_ITER`, `LINT_TOKEN_BUDGET`, `LINT_MAX_FINDINGS`); `total_cost_usd` logged 4dp; terminal status always written (I7). ✅ (test_lint.py)
- **AC-K2-2** All LLM calls via `resolve_provider_config` — no hardcoded backend, no class-name branching (I6). ✅ static test.
- **AC-K2-3** Scan reads only `pages`/`links` (no vault walk); apply touches only referencing pages; `data_version` bumped ≤1 per fix (I1). ✅
- **AC-K2-4** Human gate: scan never applies a fix; `apply` only on explicit call. Flag-only categories (orphan/contradiction/stale) acknowledge without editing human-curated content (K8). ✅
- **AC-K2-5** Frontend Lint section: Run-Lint action, findings grouped by category with Apply/Dismiss, cost line, empty state; nav slot activated; i18n EN+IT parity; vitest green; I3/I4 respected. ⏳ (in progress)
- **AC-K2-6** ADR-0037 + `docs/sequences/lint-fix.mmd` present and valid; openapi.json + ER regenerated. ✅

### F11 — Web clipper + secure clip ingress
- **AC-F11-1** Chrome MV3 extension under `extension/`: Readability + Turndown (HTML→Markdown, tables), popup with project/vault picker, posts to the local ingress.
- **AC-F11-2 (SECURITY — non-negotiable)** The local clip ingress requires a **token** (rejects unauthenticated → 401); validates **Origin/Host** against an allowlist (`chrome-extension://<id>` + loopback); caps the body size; and writes only inside the vault via a **safe-join** path check (no attacker-controlled base path). A regression test proves each: unauth→401, bad origin→reject, oversized→413, traversal→reject. (Directly answers llm_wiki audit S-1…S-6.)
- **AC-F11-3** A clipped page lands in `vault/raw/sources/` and is auto-ingested through the existing pipeline (I1/I5/K1) — no new ingest path.
- **AC-F11-4** Clip content is sanitized before render; no unbounded polling race (use the existing watcher/queue, not a naive 3 s poll that double-ingests).
- **AC-F11-5** ADR for the ingress security model; D3 sequence `web-clip.mmd`; openapi.json updated if an HTTP endpoint is added.

### F15 (rest) — PWA + Tauri v2 + CI/CD
- **AC-F15-1** PWA: web manifest + service worker; the frontend is installable; offline shell loads. No violation of I2/I3/I4.
- **AC-F15-2** Tauri v2 shell wraps the frontend as a desktop app (macOS/Windows/Linux targets configured). Close-behavior is a user setting (not hardcoded).
- **AC-F15-3 (CI HARDENING)** CI runs on `sprint/**` + `main` (fix NB-9 branch filter) and **executes the test suites** (`pytest` + `vitest`) and the **linters with pinned versions** (resolve the ruff/black drift on v0.5 files), plus the mmdc Mermaid render check (GAP-v0.5-3). Multi-OS build matrix produces installable artifacts.
- **AC-F15-4** Linter drift cleared: `ruff check` + `black --check` green tree-wide with pinned versions recorded in `pyproject.toml`/CI.

### F2 — purpose.md as provider context
- **AC-F2-1** Confirm (and complete if missing) that `purpose.md` is injected into the ingest analysis/generation prompts and the chat/query context; add a test asserting its presence in the assembled provider input.

### D1–D7 final gate
