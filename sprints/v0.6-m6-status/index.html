    5. EC-M6-HCP — what the human checkpoint requires (blocks the v1.0.0 tag)¶

    The v1.0.0 tag is intentionally not applied autonomously. Per the project DoD and because these need a real environment (TrueNAS stack + browser + OS runners), the following must be done by Emanuele (or on live CI) before tagging:

    1. Live stack smoke — docker compose up, point at Ollama/Qdrant/bge-m3, run a real ingest → verify wiki pages, graph, chat with citations, lint scan, and a real web-clip → auto-ingest.
    2. PWA — install the built frontend as an app; confirm offline app-shell loads and API calls are never served stale.
    3. Tauri desktop — let the tag-only CI matrix build the macOS/Windows/Linux binaries (or build locally with WebKit libs); launch and smoke-test the desktop window.
    4. D5 screenshots — capture the M5 + M6 pending PNGs into docs/screens/.
    5. 3-provider matrix — smoke ingest+chat on Ollama / API / CLI (I6).
    6. Sign off the four gate chains (QA / architect / tech-writer / PM) per the established process.

    On HCP pass → roll up DOCS_STATUS/BACKLOG/TRACEABILITY, bump versions to 1.0.0 (frontend/package.json, src-tauri/tauri.conf.json, backend/pyproject.toml), and tag v1.0.0.
