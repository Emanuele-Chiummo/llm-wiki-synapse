    Sprint 1 — v0.1 — Architecture & Design Note¶

    Author: solution-architect Date: 2026-06-28 Status: Design sign-off basis for EC-13 (architect gate) Source of truth: CLAUDE.md (§3 invariants, §6 architecture). Scope: docs/sprints/v0.1-scope.md. Companion ADRs: docs/adr/0001..0006. C4: docs/architecture/context.mmd, container.mmd.

    This note is the contract engineers follow for the v0.1 walking skeleton. It resolves the seven functional-analyst ambiguities, specifies the data model, and states how the design honours the invariants in force (I1, I5, I8, I9) without precluding I2/I3/I4/I6 in later sprints.

    1. Resolved ambiguities (AQ decision table)¶

    AQ Question DECISION (locked) ADR Blocks
    AQ-1 bge-m3 embedding dimension EMBEDDING_DIM is a required env var (documented default 1024); collection created with that value; validated against the live bge-m3 service at startup, fail-fast on mismatch. No integer literal in code. 0004 AC-QD-1
    AQ-2 "Unchanged file" detection mtime-then-hash: mtime equal → skip (fast path); mtime differs → sha256 over raw bytes; hash is the authoritative change signal. Never mtime-only. 0001 AC-WATCH-2, AC-K4-3
    AQ-3 Startup with pre-existing files No rescan, no DB write. Emit exactly one INFO line naming the watched dir and stating pre-existing files are not auto-indexed (I1). Not a warning. Test asserts zero rows + the INFO line. 0006 AC-WATCH-5
    AQ-4 vault_state seeding Seed on startup (idempotent), one row per vault_id (VAULT_ID env), data_version = 0 if absent; survives restart; monotonic +1 only on successful upsert ingest. 0005 AC-F16dv-1, AC-F16dv-4
    AQ-5 AC-K7-3 Obsidian open Not an architectural ambiguity — MANUAL human gate (GAP-1). Recorded in sign-off register; no pytest id. — AC-K7-3
    AQ-6 POST /ingest/trigger v0.1 response HTTP 202, body {"task_id": null, "status": "completed"\|"skipped", "page_id": "<uuid>"}. v0.2 fills task_id, adds "queued"/"running"; non-breaking superset. 0006 AC-REST-4
    AQ-7 Alembic location vs AC-PG-4 grep Alembic lives at backend/alembic/ (outside backend/app/). The no-raw-SQL grep scope is backend/app/ only, so the exclusion is implicit. — AC-PG-4

    These decisions are locked. An engineer who needs to deviate escalates to solution-architect before coding (per scope §4 and CLAUDE.md §3).

    2. Module / component design¶

    All modules are provider-free in v0.1 (no F17, no LLM, no model id anywhere — ADR-0003).

    2.1 backend/app/models.py — SQLAlchemy 2, source of truth for D2¶

    Two tables. make er introspects these models to generate docs/er/schema.mmd (I8); the ER diagram MUST match the live schema, so this file is the single contract.

    Table pages — one row per source file in vault/raw/sources/ (soft-deleted, never dropped; ADR-0005).

    Column Type Null Key / default Purpose
    id UUID no PK, default uuid4 Page identity; also the Qdrant point id (ADR-0002)
    vault_id TEXT no from VAULT_ID env Multi-vault key (single vault in v0.1)
    file_path TEXT no unique among live rows Relative path under vault/raw/sources/; join key to filesystem
    title TEXT yes NULL if absent K6 frontmatter title
    type TEXT yes NULL if absent K6 frontmatter type
    sources JSONB yes NULL if absent K6 frontmatter sources[] (array of strings)
    content_hash TEXT no — sha256 of raw file bytes; authoritative change signal (ADR-0001)
    source_mtime_ns BIGINT yes — Last seen st_mtime_ns; cheap fast-path gate (ADR-0001)
    qdrant_point_id UUID yes == id Explicit Postgres↔Qdrant join (ADR-0002)
    deleted_at TIMESTAMPTZ yes NULL = live Soft-delete marker (ADR-0005, F13)
    created_at TIMESTAMPTZ no default now() Audit
    updated_at TIMESTAMPTZ no onupdate now() Changes on every upsert (AC-WATCH-3)

    Recommended index (cheap, add in v0.1 if trivial): partial unique (vault_id, file_path) WHERE deleted_at IS NULL — enforces one live row per path (ADR-0005).

    Note vs backlog column list (AC-PG-3): source_mtime_ns is added beyond the backlog list because ADR-0001's fast path requires a stored mtime. It is a superset, not a conflict; the AC-PG-3 columns are all present. The ER diagram and the test column set must include it.

    Table vault_state — one row per vault; the data_version debounce signal for future FA2 recompute (I2). Seeded on startup (ADR-0005).

    Column Type Null Key / default Purpose
    id UUID no PK, default uuid4 Row identity
    vault_id TEXT no unique One row per vault; from VAULT_ID
    data_version INTEGER no default 0 Monotonic; +1 per successful upsert ingest (AC-F16dv-2/4)
    updated_at TIMESTAMPTZ no onupdate now() Last bump time

    provider_config is OUT of v0.1 (confirmed): no provider exists this sprint (ADR-0003), so the table would be empty schema with no reader. It enters in v0.2 with the F17 ABC.

    2.2 backend/app/watcher.py — watchdog incremental indexer (I1)¶

    • Registers a watchdog Observer on vault/raw/sources/ for *.md CREATE / MODIFY / DELETE.
    • Never enumerates the directory (no os.listdir, no glob, no rglob). Startup registers handlers only (AC-WATCH-5). This is the literal I1 guard — any directory walk in this file is a PR rejection.
    • Per event delegates to the seam: CREATE/MODIFY → ingest_file(path); DELETE → delete_file(path). The watcher contains no Postgres/Qdrant/embedding code (ADR-0003).
    • Change gate (ADR-0001) lives in the seam, not the handler, so POST /ingest/trigger shares it.
    • Writes nothing under raw/ (AC-K1-5); the only file the ingest path writes is wiki/log.md (append) plus DB/vector stores.

    2.3 backend/app/ingest/orchestrator.py — the thin seam (ADR-0003, I6)¶

    Single entry points used by BOTH watcher and REST:

    • ingest_file(file_path) -> IngestResult — mechanical only in v0.1: read bytes → mtime/hash gate → parse frontmatter (K6, tolerant: missing → NULL + warning, AC-K6-2/3) → persist_metadata (Postgres upsert) → upsert_vector (embed via EMBEDDING_URL + Qdrant upsert) → append_log (one INDEXED line) → bump_version. Returns {page_id, status: "completed"|"skipped"}.
    • delete_file(file_path) — set deleted_at, remove Qdrant point.
    • Helpers persist_metadata / upsert_vector / append_log / bump_version are factored so v0.2's orchestrated loop reuses them as validate / write_page primitives.
    • Marked extension point (comment + no code): where v0.2 resolves a provider from provider_config, calls capabilities(), and branches orchestrated-loop vs delegated-CLI. v0.1 leaves it empty (I6 preserved; ADR-0003).

    2.4 Qdrant integration¶

    • Collection synapse_pages, vector size = EMBEDDING_DIM (ADR-0004), distance Cosine.
    • Created on startup if absent; if it exists with a different size, fail-fast (no silent recreate, ADR-0004).
    • Point id == pages.id. Payload = {file_path, title, type} (AC-QD-2). Upsert replaces in place (no orphan, AC-WATCH-3); delete by id (AC-WATCH-4 / AC-QD-3).
    • Embeddings fetched ONLY from EMBEDDING_URL (I9); no local model (AC-QD-4 / AC-WATCH-6).

    2.5 backend/app/main.py — FastAPI (4 endpoints)¶

    Method Path Behaviour AC
    GET /status 200; {data_version, started_at, uptime} AC-REST-1, AC-F16dv-3
    GET /pages 200; paginated (limit/offset), deleted_at IS NULL AC-REST-2
    GET /pages/{id} 200 full metadata; 404 unknown; 422 malformed UUID AC-REST-3, AC-REST-6
    POST /ingest/trigger sync ingest_file, 202, body per AQ-6; 422 on bad body AC-REST-4, AC-REST-6
    • Startup sequence (ordered): ensure vault/wiki/.obsidian/app.json (I5, AC-K7-1) → seed vault_state (ADR-0005) → validate EMBEDDING_DIM vs live service + ensure synapse_pages (ADR-0004) → start watcher Observer → emit AQ-3 INFO line if raw/sources/ non-empty.
    • OpenAPI auto-served at /openapi.json; make openapi snapshots it to docs/api/openapi.json (D4, AC-D4-*).

    2.6 Config (env vars, no secrets in code — AC-DC-5)¶

    DATABASE_URL, QDRANT_URL, EMBEDDING_URL, EMBEDDING_DIM (required), VAULT_ID, VAULT_PATH. Defaults in .env.example + docker-compose.yml; .env gitignored.

    3. Invariant-compliance statement¶

    Inv In force? How v0.1 honours it Verifying AC
    I1 Incremental only YES (core) Watcher reacts to events only, never walks the dir; mtime-then-hash skip (ADR-0001); duplicate drop → no row/point/log/version change. AC-WATCH-1..5, AC-K4-3
    I2 Server-side cached layout Not exercised (no graph) data_version shaped now as the future debounce signal; seeded + monotonic so v0.3 can rely on it. No decision precludes server-side FA2. AC-F16dv-*
    I3 No per-token chat work Not exercised (no chat) No chat path exists; nothing precludes stream-end parsing later. —
    I4 CodeMirror, virtualised Not exercised (no UI) No frontend in v0.1; no editor choice baked. —
    I5 Obsidian compatibility YES .obsidian/app.json auto-written on startup; all service-written wiki/*.md (index.md, log.md) carry valid YAML frontmatter; raw/ never written. AC-K7-1/2/3, AC-K1-5
    I6 Pluggable inference Not exercised, protected Zero provider/model id/LLM call in v0.1; ingest routed through a thin seam with a marked extension point so v0.2 slots F17 without touching callers (ADR-0003). design / EC-13
    I7 Bounded loops Not exercised (no loops) No loop in v0.1; the seam's empty branch is where v0.2's max_iter+token_budget loop lands; straight-line body imposes no loop assumptions. —
    I8 Docs-as-DoD YES D1 hand-authored C4 (context+container); D2 generated by make er from models.py (must match live schema); D4 by make openapi from FastAPI. ADRs 0001–0006 committed (D7 early, allowed). AC-D1/D2/D4-*, AC-DC-3/4
    I9 Do not reinvent YES Qdrant + bge-m3 reused via QDRANT_URL / EMBEDDING_URL; no new embedding/vector container in compose (only synapse-backend + postgres16); no Tavily; no local model. AC-QD-*, AC-WATCH-6, AC-DC-1

    4. EC satisfiability — flags¶

    The design satisfies all 15 M1 exit criteria with the following explicit notes (no EC is un-satisfiable, but three carry conditions):

    EC Status Note
    EC-1 Satisfiable Includes AC-K7-3 = MANUAL human gate (open in Obsidian). Not automatable; recorded in sign-off register (GAP-1 / AQ-5).
    EC-2 Satisfiable Hinges on ADR-0001 mtime-then-hash; "modify → updated_at changes + 1 log line" requires editor save to change mtime (it does).
    EC-3..EC-7 Satisfiable Covered by ADR-0001/0005/0006; data_version monotonic by construction.
    EC-5 Conditional AC-QD-1 dimension must be confirmed against the running bge-m3 at startup (ADR-0004). If the live service is unreachable at CI time, the dimension-validation test needs a mock endpoint returning a vector of EMBEDDING_DIM length (see GAP-4). The collection-creation value itself is never hardcoded.
    EC-8 Conditional docker compose up starts only synapse-backend + postgres16; Qdrant/bge-m3 are external (I9). AC-DC-1's full integration therefore needs reachable TrueNAS services OR mocked URLs — handled by GAP-4's split unit/integration CI stages. Not a design defect; a test-environment condition.
    EC-9 Satisfiable C4 context+container authored; mmdc render check (AC-D1-1/2) must run in CI — not validated on the architect's machine (mmdc not installed locally). Flagged to devops for the docs CI stage.
    EC-10 Satisfiable make er must emit source_mtime_ns too (superset note in §2.1); the live-schema match test must expect it.
    EC-11..EC-15 Satisfiable Human/agent gates; no architectural blocker.

    No EC is impossible to satisfy. The conditions on EC-5 / EC-8 / EC-9 are test-environment matters (external service reachability, CI tooling), already anticipated by the QA gap register (GAP-4) — they are called out so devops/QA wire the CI stages accordingly, not because the design cannot meet them.

    5. Sign-off basis (EC-13)¶

    Architect approves the v0.1 design (models.py, watcher.py, main.py, ingest seam, Qdrant integration) on the basis above, conditional on engineers honouring: the thin-seam rule (ADR-0003), the no-directory-walk rule (I1/ADR-0001), the no-hardcoded-dimension rule (ADR-0004), and the make er superset including source_mtime_ns. PR review enforces these.
