    Sprint 3 — v0.3 — Architecture & Design (F4 graph engine + I2 cache)¶

    Milestone: M3 — "Graph live, no main-thread freeze" Author: solution-architect · Date: 2026-06-28 Resolves: AQ-v0.3-1..7 · ADRs: 0012, 0013, 0014, 0015 Invariants owned this sprint: I2 (headline), I4, I1, I3 (pre-compliance)

    1. AQ decision table (the 7 questions)¶

    AQ P Question DECISION ADR
    AQ-v0.3-1 P0 Edge-weight combining formula ADDITIVE over undirected page-pair: w = 3·direct_link_count + 4·shared_source_count + 1.5·adamic_adar + 1·same_type. Each term defined exactly (count / set-intersection / igraph AA / 0-1 indicator). Edge persisted iff w>0. 0012
    AQ-v0.3-2 P1 FA2 determinism Fixed seed = 42 (GRAPH_LAYOUT_SEED override). Identical topology+weights → identical coords. Determinism assertion added to test_graph_engine.py. 0013
    AQ-v0.3-3 P0 GET /graph sync 200 vs async 202 Synchronous 200. Hit → read coords (no FA2). Miss → one inline recompute then return. Never 202. (<2s for <500 nodes.) 0014
    AQ-v0.3-4 P1 Incremental semantics "Incremental" = (a) one ingest mutates only that page's rows, (b) recompute is debounced (once per window). FA2 is global → existing coords MAY change; AC-F4-9 asserts row-level incrementality, NOT coord stability. 0013
    AQ-v0.3-5 P0 edges table persisted? Yes, persisted in Postgres (edges table, written each recompute, read by GET /graph). Not computed per-request. 0012, 0014
    AQ-v0.3-6 P1 Coord storage Columns on pages (pages.x, pages.y DOUBLE PRECISION NULL). Not a join table. Matches BACKLOG. Coord write stays inside the page-row upsert (I1). 0013
    AQ-v0.3-7 P1 Cache trigger/detection In-process debounce on data_version bump (notify_bump() from ingest) + polling-reconcile fallback; injectable clock+tick for testing; default 5s window. No LISTEN/NOTIFY. 0014

    AQ-1 follow-up note to functional-analyst (one-line fixture fix)¶

    Per ADR-0012, two same-type pages get the +1 type term. v0.3-stories §1 expects edge P3–P5 weight 0 / absent, but P3 and P5 are both concept → the formula gives weight 1.0 (type only). Fixture correction: either set P5's type ≠ P3's type, or assert P3–P5 weight == 1.0. The formula (ADR-0012) is the source of truth; the QA fixture adjusts. The lower-bound assertions for P1–P2 (≥11), P1–P4 (≥8), P2–P4 (≥5) are unaffected.

    2. Locked edge-weight formula (F4 / AC-F4-1)¶

    For an unordered pair of live pages (A, B) (graph = pages where deleted_at IS NULL):

    weight(A, B) =  3.0 · direct_link_count(A, B)      # resolved links A↔B in either direction (0,1,2)
             +  4.0 · shared_source_count(A, B)     # |set(A.sources) ∩ set(B.sources)|
             +  1.5 · adamic_adar(A, B)             # igraph similarity_inverse_log_weighted, undirected resolved-link graph
             +  1.0 · same_type(A, B)               # 1 iff A.type == B.type AND both non-NULL, else 0
    • direct_link_count: count of non-dangling, resolved links rows with {source_page_id, target_page_id} == {A,B}; each direction counts 1.
    • shared_source_count: intersection size of the pages.sources JSONB arrays (NULL → ∅).
    • adamic_adar: Σ_{c ∈ N(A)∩N(B)} 1/ln(deg(c)) on the undirected unweighted graph built from resolved links; via igraph (I9), no hand-rolled math.
    • same_type: NULL type does NOT match NULL (unknown ≠ a type).
    • Persist iff weight > 0 (sparse table; an AA-only edge between non-linked pages IS kept).

    This is deterministic and FA2-seed-independent → byte-stable QA fixtures. Full term table and worked example in ADR-0012.

    3. Data-model deltas (D2)¶

    3.1 pages — two new columns¶

    Column Type Null Notes
    x DOUBLE PRECISION YES FA2 x-coordinate; NULL until first layout (ADR-0013)
    y DOUBLE PRECISION YES FA2 y-coordinate; NULL until first layout

    3.2 edges — new table (ADR-0012 / AQ-5)¶

    Column Type Null Notes
    id uuid PK NO gen_random_uuid()
    vault_id string NO scope edges per vault (matches pages/vault_state pattern)
    source_page_id uuid FK → pages.id NO unordered pair, stored canonically (smaller uuid first)
    target_page_id uuid FK → pages.id NO
    weight DOUBLE PRECISION NO ADR-0012 additive weight (> 0)
    signals JSONB YES per-signal breakdown {direct, source, aa, type} for audit/debug (OPTIONAL but recommended; enables AC-F4-1(e) independent-signal assertions without recomputation)
    created_at timestamptz NO now()

    Constraints: unique (vault_id, source_page_id, target_page_id) on the canonicalised pair; indexes on source_page_id and target_page_id for GET /graph reads and cascade cleanup (F13, v0.5). Recompute replaces this vault's rows (delete-then-insert or upsert) in one txn.

    4. GraphEngine algorithm (backend/app/graph/engine.py)¶

    recompute(vault_id) — single bounded pass (I7), reads only Postgres (I1, never walks vault):

    1. SELECT id, page_type, sources FROM pages WHERE vault_id=? AND deleted_at IS NULL → nodes.
    2. SELECT source_page_id, target_page_id FROM links WHERE dangling=false AND target_page_id IS NOT NULL → resolved directed edges.
    3. Build undirected unweighted igraph; compute Adamic-Adar (similarity_inverse_log_weighted).
    4. For each candidate pair (sharing a source OR a neighbour OR a direct link OR same type), compute the 4-signal additive weight (ADR-0012); keep weight > 0.
    5. Build the weighted igraph; run seeded FA2 (seed=42, weights as edge weights) → coords.
    6. One transaction: batch UPDATE pages SET x,y WHERE id=?; replace edges rows.
    7. Stamp the cache marker with the current data_version; log node/edge count + duration (I7).

    Complexity: N nodes, L resolved links, P candidate pairs (≪ N²; only neighbour/source/ type-sharing pairs). Weights O(L+P); FA2 O(iters·(N+E)); persistence O(N+E). Sub-2s for the target <500-node vault → supports synchronous GET /graph.

    Incremental-friendliness (I1): the engine consumes DB tables that the incremental indexer already maintains; it issues column-level coord upserts, not a full-table rewrite. One ingest = +1 page row + its links; the global relayout that follows is debounced and runs once per window (ADR-0013 §4).

    5. GraphCache mechanism (backend/app/graph/cache.py) — the I2 cache¶

    • State: persisted coords (pages.x/y + edges) tagged by a marker = the data_version they were computed from. hit ⇔ vault_state.data_version == marker.
    • Trigger: notify_bump() from the ingest path after a data_version +1 (in-process), plus a polling-reconcile fallback. No LISTEN/NOTIFY.
    • Debounce: on bump set fire_at = clock.now() + window (default 5s). A bump within the window resets fire_at (burst collapse → one recompute, AC-F16db-2).
    • Bounded queue (I7): at most one recompute in-flight; a bump during a run sets a single pending flag → exactly one follow-up after completion (AC-F16db-3). Queue depth ≤ 1.
    • Testability (no sleeps): injectable clock.now() + a tick() that fires when clock.now() >= fire_at and no run is in-flight. QA advances the fake clock and drives tick(); asserts recompute call counts (1 for AC-F16db-1/2; 2 for AC-F16db-3).
    • GET /graph hit: pure read, cached:true, X-Graph-Cache: hit, no FA2.
    • GET /graph miss: one inline synchronous recompute (cold start / stale), cached:false, X-Graph-Cache: miss. Shares the in-flight guard with the background path → no concurrent FA2.

    Full state machine and the in-flight/pending rules: ADR-0014.

    6. GET /graph API contract (D4 / AC-F4-3, AC-F4-4, AC-D4v3-1)¶

    GET /graph → 200 synchronous. Header X-Graph-Cache: hit|miss.

    {
  "nodes": [
    {"id": "uuid-string", "title": "string", "type": "string|null",
     "x": 0.0, "y": 0.0, "size": 1.0, "degree": 3}
  ],
  "edges": [
    {"source": "uuid-string", "target": "uuid-string", "weight": 11.0}
  ],
  "data_version": 7,
  "cached": true
}
    • nodes: one per live page with non-NULL x/y. Required: id, title, type, x, y. Optional rendering hints (derived, not persisted): size (monotonic in degree, default 1.0), degree (incident-edge count).
    • edges: one per undirected edges row with live endpoints. Required: source, target, weight.
    • data_version (int): the version the coords correspond to. cached (bool): mirrors the header.

    AC-F4-3 asserts the required core fields + types. AC-F4-4 asserts the second same-version call returns cached:true + X-Graph-Cache: hit with no second FA2 (mock recompute count == 1).

    7. sigma.js client contract (I2 / I4 / I3 — ADR-0015)¶

    • Single read-only route (/). React 19 + Vite + TS strict; sigma.js (WebGL) over graphology.
    • On mount: one fetch('/graph') → pure transform → graph.addNode(id,{x,y,label,type,size}) with server coords + graph.addEdge(source,target,{weight}). sigma renders into ONE <canvas>.
    • HARD RULE (P0 block): no graphology-layout-forceatlas2, ForceAtlas2, d3-force, forceSimulation, @antv/layout, or any rAF loop that mutates node x/y — in source or dist/. Enforced by the bundle-grep test (AC-FE-2) and an architect bundle review before the G2 test is written. Any client-layout PR is escalated and blocked.
    • Interaction: node click → GET /pages/{id} → read-only title/type tooltip. No editor (no CodeMirror, I4), no chat, no provider selector (v0.4).
    • Zustand (I3): selector subscriptions + shallow equality; no whole-store subscription; pure API→graphology transform unit-tested in isolation (AC-FE-5).
    • Lists: none in v0.3. Any future node/search list > 50 items MUST use TanStack Virtual (I4).
    • G2 by construction: no client layout → only one fetch + a small pure transform + GPU draw; no main-thread long task > 50ms.
    • G4 by construction: all nodes drawn in one WebGL canvas → DOM count in the graph container < 20 regardless of size; precomputed coords → pure GPU redraw → ≥60fps for 200 nodes/500 edges.

    I1 GraphEngine reads pages+links tables only — never walks vault/. Coord write is a column-level upsert on existing rows. One ingest mutates only that page's rows; relayout is debounced, not per-file. (ADR-0013)
    I3 Zustand selectors + shallow equality, pure transform — pre-compliance for v0.4 chat. (ADR-0015)
    I5 Graph state is Postgres-only (x/y columns + edges table). No .md/frontmatter writes by the engine. test_obsidian_check.py (15) stays green. (ADR-0013)
    I7 recompute is a single pass; cache queue bounded to one in-flight + one pending; bumps during a run collapse to one follow-up. Engine logs node/edge count + duration. (ADR-0014)

    9. EC coverage / risks¶

    All EC-M3-1..15 are satisfiable by this design. No EC is un-satisfiable. Two items to flag:

    • EC-M3-1 / AC-F4-1 fixture (P3–P5): the stories fixture expects weight 0 for two same-type pages, but the locked formula gives +1 (type). Requires the one-line QA fixture correction in §1 (set P5 type ≠ P3, or assert weight == 1.0). Not a design defect — a fixture/formula reconciliation. Flagged to functional-analyst.
    • EC-M3-3 cold-start miss latency: a first-ever GET /graph runs FA2 inline (<2s for target sizes). If a real vault grows beyond ~1–2k nodes, the synchronous miss could exceed a comfortable request budget; the background debounce keeps the common case a hit. Revisit async/202 only if measured latency demands it (out of v0.3 scope; explicitly deferred).

    No invariant is traded for convenience in this design.

    10. F4 / I2 SIGN-OFF¶
