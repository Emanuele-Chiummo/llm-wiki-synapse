    Synapse v0.6 — Runbook di test (per Emanuele)¶

    Obiettivo: allineare il tuo Mac a GitHub e testare TUTTO ciò che è entrato in M6/v0.6. Branch di riferimento: sprint/v0.6 — HEAD cb55457, 14 commit sopra sprint/v0.5. Nota: questo runbook gira sul TUO Mac (io sono in un container remoto e non vedo /Users/...).

    0. Allinea il locale a GitHub¶

    Il tuo checkout è in [project-root] (path con spazi → usa gli apici).

    cd "[project-root]"

git fetch origin --prune
git status                      # committa/stasha eventuali modifiche locali PRIMA
git checkout sprint/v0.6
git pull --ff-only origin sprint/v0.6

    Verifica di essere allineato (deve dire 0 / 0): 

    git rev-list --left-right --count origin/sprint/v0.6...HEAD   # atteso: 0   0
git log --oneline -1                                          # atteso: cb55457 feat(settings): Web Clipper + Web Search ...

    Se hai lavoro locale che vuoi buttare per allinearti esattamente a GitHub: 

    git reset --hard origin/sprint/v0.6      # ⚠️ distrugge le modifiche locali non committate

    Migrations attese nel repo (fino alla 0016): 0014_lint_tables · 0015_vault_state_clip_config · 0016_vault_state_web_search_config.

    1. Prerequisiti¶

    • Docker Desktop (per Postgres + backend)
    • Node.js 20+ e npm (frontend)
    • Python 3.11 (solo se vuoi girare i test backend fuori da Docker)
    • Rust (rustup) — solo per il build desktop Tauri (su macOS il WebKit è di sistema, non servono librerie extra)
    • Servizi già sul tuo TrueNAS/Tailscale: Ollama (+ bge-m3), Qdrant, SearXNG

    2. Configura l'ambiente¶

    cp .env.example backend/.env      # oppure ./.env, a seconda di come monti il compose
    Nel .env punta ai TUOI servizi (localhost / Tailscale / host.docker.internal): 
    DATABASE_URL=postgresql+asyncpg://synapse:synapse@postgres:5432/synapse
QDRANT_URL=http://host.docker.internal:6333
EMBEDDING_URL=http://host.docker.internal:11434/api/embeddings
EMBEDDING_MODEL=bge-m3
EMBEDDING_DIM=1024
OLLAMA_URL=http://host.docker.internal:11434
# opzionale ora: SEARXNG e CLIP si possono configurare anche dalle Impostazioni (UI)
# SEARXNG_URL=http://host.docker.internal:8888
VAULT_ID=default
VAULT_PATH=/vault

    CLIP_* e LINT_* NON sono obbligatori nel .env: il clipper è disattivo di default e si configura da Impostazioni → Web Clipper; SearXNG si può impostare da Impostazioni → Web Search.

    3. Avvia lo stack (un solo comando) — consigliato per il Mac¶

    Synapse è client/server: l'UI (web/PWA/desktop) è un client sottile, il motore è il backend + Postgres + Qdrant + Ollama. Quindi "aprire l'app" da solo non basta: serve lo stack su. Sul tuo Mac usa lo stack completo in un comando (frontend + backend + postgres + qdrant):

    make dev            # = docker compose -f docker-compose.yml -f docker-compose.dev.yml up
    Requisito unico: Ollama attivo a http://localhost:11434 (Ollama.app). Il container esegue da solo alembic upgrade head (quindi applica anche 0014/0015/0016).

    Poi apri http://localhost:5173 (UI). API su http://localhost:8000. 

    curl -s localhost:8000/status | jq          # health check backend
    Stop: make dev-down.

    Alternativa "produzione TrueNAS" (backend + postgres in container; Qdrant/Ollama/SearXNG esterni; niente frontend containerizzato → il frontend lo servi a parte): make up + make logs + make migrate, e per l'UI cd frontend && npm ci && npm run dev (proxy → :8000, override con BACKEND_PROXY_TARGET).

    Verifica migration applicate (deve arrivare a 0016): 

    docker compose -f docker-compose.yml -f docker-compose.dev.yml exec synapse-backend alembic current

    Infine in UI: Impostazioni → LLM Models → aggiungi un provider (Local Ollama / API / CLI) — senza, ingest e chat non partono.

    4. Usare il tuo abbonamento Claude (provider CLI, costo $0) — ADR-0042¶

    Il provider CLI (claude-agent-sdk) può girare sul tuo abbonamento Pro/Max invece che a consumo con una API key. Funziona anche dentro il container, via un token OAuth a lunga durata.

    Passo 1 — genera il token (una volta, sul Mac loggato con l'account Pro/Max): 

    claude setup-token          # stampa un token OAuth (sk-ant-oat01-...), valido ~1 anno — copialo

    Passo 2 — mettilo nel .env e NON impostare la API key: 

    CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-...     # il token del passo 1
# ANTHROPIC_API_KEY  ← lascialo NON impostato (nemmeno vuoto)!

    ⚠️ Trappola di precedenza: se ANTHROPIC_API_KEY è impostata (anche =""), vince lei e ti fa pagare a token. Per l'abbonamento deve essere completamente assente.

    Passo 3 — riavvia lo stack (make dev): il container legge CLAUDE_CODE_OAUTH_TOKEN, il claude CLI incluso nell'SDK autentica via abbonamento. Nell'ingest/chat il total_cost_usd sarà $0.00.

    Passo 4 — in UI: Impostazioni → LLM Models → aggiungi un provider CLI e scegli il modello (es. claude-opus-4-8). Fai un ingest o una chat: costo $0 nell'Activity panel.

    I tre modi di autenticazione (ne scegli uno), documentati anche in .env.example:

    Variabile Provider Costo
    ANTHROPIC_API_KEY API / CLI a token
    CLAUDE_CODE_OAUTH_TOKEN CLI (abbonamento, in container) ✅ $0
    CLAUDE_CODE_USE_SUBSCRIPTION=true CLI (login host/montato) $0

    Nota Tauri: il guscio desktop resta un client HTTP verso localhost:8000 — non cambia nulla per l'auth. L'abbonamento è deciso dal backend, non da Tauri.

    5. Test automatici (attesi verdi)¶

    Backend (venv locale): 

    cd backend
python -m venv .venv && . .venv/bin/activate
pip install -e ".[dev]"
python -m pytest -q          # atteso: 1018 passed, 4 skipped
ruff check app tests && black --check app tests && mypy app   # tutto clean
    Frontend: 
    cd frontend
npm run test                 # atteso: 742 passed
npx tsc --noEmit && npm run lint    # clean

    6. Smoke manuale (EC‑M6‑HCP) — checklist¶

    Spunta ognuna nel browser/app:

    Core (Ingest · Query · Lint) - [ ] Crea progetto → Sources: importa un documento → parte l'ingest (Activity panel). - [ ] Compaiono pagine wiki, entità/concetti; Graph mostra i nodi/archi. - [ ] Chat: fai una domanda → risposta con citazioni [n]. - [ ] Lint (nuovo): sezione Lint → Run Lint → compaiono findings (orphan/missing‑xref/contradiction/stale/missing‑page) → prova Fix (missing‑xref/missing‑page) e Acknowledge (gli altri) e Dismiss.

    Web Search (nuovo pannello) - [ ] Impostazioni → Web Search: imposta l'URL SearXNG (es. http://host.docker.internal:8888), categorie, max query → salva; il badge "source" passa a db. - [ ] Deep Research su un gap → esegue query SearXNG → sintetizza → auto‑ingest.

    Web Clipper (nuovo pannello + estensione) - [ ] Impostazioni → Web Clipper: Enable ON → Genera token (copia il valore mostrato una‑tantum). - [ ] Carica l'estensione: Chrome → chrome://extensions → Modalità sviluppatore → Carica non pacchettizzata → seleziona la cartella extension/ → copia l'ID dell'estensione. - [ ] In Impostazioni → Web Clipper, aggiungi chrome-extension://<ID> in Allowed origins → salva. - [ ] Nell'estensione (Options): base URL http://localhost:8000, incolla il token. - [ ] Vai su una pagina web → clicca l'estensione → Clip → il markdown finisce in vault/raw/sources/ e viene auto‑ingerito (lo vedi nell'Activity panel). - [ ] Prova gli errori attesi: token errato → 401; se disabiliti l'ingress → 503.

    Altro - [ ] Review queue: un ingest genera proposte → Approve / Deep‑Research / Skip. - [ ] Cascade delete: elimina una source → la sua pagina wiki e i wikilink morti spariscono; entità condivise restano.

    PWA - [ ] cd frontend && npm run build && npm run preview → apri l'URL → in Chrome Installa app → verifica shell offline; le chiamate API NON sono cachate (nessun dato stantìo).

    Desktop (Tauri) - [ ] cd frontend && npm run tauri:build (serve Rust) → installer in src-tauri/target/release/bundle/… → lancia l'app. - [ ] In alternativa: npm run tauri:dev per la finestra desktop in sviluppo.

    Matrice 3 provider (I6) - [ ] Ripeti un ingest+chat con provider Local (Ollama), API, CLI.

    7. Se qualcosa non va — check rapidi¶

    Sintomo Verifica
    /status non risponde make logs; Postgres su? docker compose ps
    Ingest non parte provider configurato in Impostazioni → LLM Models? EMBEDDING_URL/OLLAMA_URL raggiungibili?
    Deep Research 503 SearXNG impostato (Impostazioni → Web Search o SEARXNG_URL)?
    Clip 401 token corretto nelle Options dell'estensione? Enable ON?
    Clip 403 chrome-extension://<ID> negli Allowed origins?
    Migrazioni mancanti docker compose exec synapse-backend alembic upgrade head
    Frontend non chiama il backend proxy: backend su :8000? BACKEND_PROXY_TARGET giusto?

    8. Quando lo smoke è OK → verso v1.0.0¶

    Superata la checklist §6 (questa È l'EC‑M6‑HCP), il tag è meccanico: 1. bump versione a 1.0.0 in frontend/package.json, src-tauri/tauri.conf.json, backend/pyproject.toml; 2. cattura gli screenshot D5 in docs/screens/; 3. git tag v1.0.0 && git push origin v1.0.0 (fa scattare il job CI build multi‑OS Tauri).

    Dimmi "vai" e faccio io i bump + il tag quando avrai dato l'OK allo smoke.
