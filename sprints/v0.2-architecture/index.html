    Sprint 2 — v0.2 — Architecture & F17 Sign-off¶

    Owner: solution-architect Milestone: M2 — "Agentic loop closed, 3 providers" Date: 2026-06-28 Status: SIGNED OFF — APPROVED to implement (see §9) Authority: this document is the binding design for v0.2. Engineers implement to it; PRs are reviewed against it. ADR-0007..0011 carry the rationale.

    This is the design sign-off for F17 (InferenceProvider abstraction + capability-aware routing) and the bounded ingest loop. It resolves the 8 functional-analyst questions (AQ-v0.2-1..8), locks the contract schemas, specifies the data model and the routing/loop, defines the MCP tool contracts, and states invariant compliance. It does NOT contain provider implementation code (that is ai-agent-engineer's work).

    1. AQ decision table (the 8 blocking questions — RESOLVED)¶

    AQ Question Decision Recorded in
    AQ-v0.2-1 analyze() re-run on retry? analyze ONCE per run; only generate() is retried with validation errors appended to retrieval_context. analyze.call_count==1, generate.call_count==max_iter on non-convergence. ADR-0007 §4
    AQ-v0.2-2 ingest-run logging: structlog vs Postgres table? Postgres ingest_runs table (system of record for cost auditing per I7/§11) PLUS a structured log line for live tail. ADR-0008 §4
    AQ-v0.2-3 exact Pydantic schemas for Analysis/WikiPage/Message/ProviderCapabilities? Locked in backend/app/ingest/schemas.py; Pydantic v2 for parsed DTOs, frozen dataclass for descriptors. Full field tables in §2 below. ADR-0011, §2
    AQ-v0.2-4 per-backend token accounting? Uniform Usage{input_tokens, output_tokens, total_cost_usd}; each provider maps native fields. Anthropic usage.*; Ollama prompt_eval_count/eval_count; CLI cost=$0.00; Local cost=$0.00. ADR-0009 §2
    AQ-v0.2-5 provider_config "operation" parameterisation? Add a separate nullable operation column (ingest|chat|lint); scope ∈ global|vault|operation. Resolution: operation+vault > vault > global. ADR-0008 §2
    AQ-v0.2-6 MCP transport? stdio for v0.2 (CliAgentProvider native path); HTTP deferred to v0.4. Test client exercises tools in-process. ADR-0010 §1
    AQ-v0.2-7 WikiPage "invalid" threshold + retry granularity? Invalid if ANY page fails: type∈enum, non-empty title, non-empty sources[] (incl. origin path), lang, non-empty content. Whole-batch retry. Dangling links do NOT invalidate. ADR-0007 §5
    AQ-v0.2-8 $1/run cost-anomaly WARNING site? Inline in orchestrator, after the ingest_runs row/log line is written. Sets cost_anomaly=True. ADR-0009 §3

    2. Locked schemas (the contract — backend/app/ingest/schemas.py)¶

    Normative. ai-agent-engineer implements these exactly. R = required, O = optional. Pydantic v2 BaseModel unless marked dataclass.

    PageType (str Enum)¶

    entity | concept | source | synthesis | comparison (plus overview and index reserved for auto-generated catalogue pages — NOT valid suggested_pages types; used only by K3/F3 writers.)

    Analysis (BaseModel)¶

    Field Type R/O Constraint
    topics list[str] R len ≥ 1
    entities list[str] R len ≥ 0 (may be empty)
    language str R ISO-639-1, e.g. "en", "it"
    suggested_pages list[SuggestedPage] R len ≥ 1
    summary str O short abstract for overview generation

    SuggestedPage (BaseModel)¶

    Field Type R/O Constraint
    title str R non-empty
    type PageType R in enum
    rationale str O why this page

    WikiFrontmatter (BaseModel, extra="allow")¶

    Field Type R/O Constraint
    type PageType R in enum
    title str R non-empty
    sources list[str] R non-empty; contains the origin relative path (F3)
    lang str R ISO-639-1, == Analysis.language
    (extra keys) Any O allowed for future frontmatter

    WikiPage (BaseModel)¶

    Field Type R/O Constraint
    title str R non-empty (mirrors frontmatter.title)
    type PageType R in enum (mirrors frontmatter.type)
    content str R non-empty Markdown body, frontmatter block stripped
    frontmatter WikiFrontmatter R typed; enforces I5 at the schema boundary

    Message (BaseModel) — used only by stubbed chat()¶

    Field Type R/O Constraint
    role Literal["user","assistant","system"] R backend-neutral (I6)
    content str R

    ProviderCapabilities (frozen dataclass)¶

    Field Type Constraint
    mode Literal["local","api","cli"]
    supports_tools bool
    supports_agentic_loop bool the ONLY routing signal (I6)
    max_context int tokens
    name str non-empty (audit only, never routing)

    Usage (frozen dataclass)¶

    Field Type Constraint
    input_tokens int ≥ 0
    output_tokens int ≥ 0
    total_cost_usd float 0.0 for local & cli

    3. InferenceProvider ABC (backend/app/ingest/provider/base.py)¶

    class InferenceProvider(abc.ABC):
    @abstractmethod
    async def analyze(self, source_text: str, vault_context: str) -> Analysis: ...
    @abstractmethod
    async def generate(self, analysis: Analysis, retrieval_context: str) -> list[WikiPage]: ...
    @abstractmethod
    async def chat(self, messages: list[Message], retrieval_context: str) -> AsyncIterator[str]: ...
    @abstractmethod
    def capabilities(self) -> ProviderCapabilities: ...

    Rules (enforced in review + CI): - chat() is implemented on all three backends but raises NotImplementedError immediately (no network call) in v0.2. Signature locked now → non-breaking body change at v0.4 (F6). - Routing reads capabilities().supports_agentic_loop only. NEVER isinstance, type(...), class-name strings, or provider_type == "cli". (AC-K2-4, ADR-0007 §3.) - No model id / API key / endpoint URL appears outside backend/app/ingest/provider/ (AC-F17-4/8, ADR-0008 §3). Keys come from env inside api.py/cli.py only. - Usage is recorded out-of-band onto a run-scoped accumulator the orchestrator passes in; it is NOT part of the public return types (keeps WikiPage clean; lets the CLI path share the same accounting surface).

    Capability matrix (AC-F17-3)¶

    Provider mode supports_agentic_loop supports_tools route
    OllamaProvider local False model-dependent orchestrated
    ApiProvider api False True orchestrated
    CliAgentProvider cli True True delegated

    4. Capability-aware routing + bounded loop (orchestrator.py)¶

    Inserted at the ADR-0003 extension point inside ingest_file(); callers (watcher, REST) unchanged.

    resolve provider from provider_config (operation="ingest")   # ConfigResolver, ADR-0008
caps = provider.capabilities()
if caps.supports_agentic_loop:            # CLI
    result = delegate_ingest(provider, source_text, vault_context, token_budget_cli)
else:                                     # Local / API — ORCHESTRATED LOOP
    analysis = await provider.analyze(source_text, vault_context)   # ONCE (AQ-1)
    ctx = retrieval_context
    for i in range(1, max_iter+1):                                  # I7 bound #1
        if accumulator.tokens >= token_budget: break                # I7 bound #2 (pre-call)
        pages = await provider.generate(analysis, ctx)
        errors = validate(pages)                                    # ADR-0007 §5
        if not errors:
            converged = True; break
        ctx = augment(ctx, errors)                                  # retry generate only
    write pages, parse links (K5), update index.md (K3) / overview.md (F3)
finalize accumulator → total_tokens, total_cost_usd
INSERT ingest_runs{...converged...}                                 # I7 log (ADR-0008 §4)
if total_cost_usd > 1.00: log WARNING; set cost_anomaly=True        # inline (AQ-8)

    I7 bounds (every loop)¶

    Loop max_iter token_budget cost logged fallback
    Orchestrated ingest 3 60 000 ingest_runs + log n/a
    CLI delegated n/a (provider-internal) 100 000 (passed to SDK; abort on exceed) ingest_runs (cost=$0.00) n/a
    Provider fallback 1 attempt — records fallback provider_name, converged=False exactly once, then surface IngestError

    Cost accounting points¶

    1. After analyze() → add Usage to accumulator.
    2. After each generate() → add Usage.
    3. Before each generate() → check token_budget (stop if exceeded).
    4. At run end → finalize → write ingest_runs → inline anomaly check.

    5. Data-model deltas (for D2 / make er)¶

    All three tables ship in ONE Alembic migration (scope §2 K3/K5 decision). Full column tables in ADR-0008. Summary:

    • provider_config: id, scope (global|vault|operation), operation (ingest|chat|lint, null), vault_id (null at global), provider_type (local|api|cli), model_id, base_url (null), max_iter (default 3), token_budget, is_fallback (default false), created_at, updated_at. No API key column (§12 — env only). Seeded with one global row using §12 model ids via the Alembic data migration (never literals in app code, AC-F17-8).
    • ingest_runs: id, vault_id, page_id (FK pages, null), provider_name, provider_type, model_id, route (orchestrated|delegated), max_iter_used, total_tokens, total_cost_usd (numeric(10,4)), converged, cost_anomaly (default false), started_at, finished_at.
    • links (K5): id, source_page_id (FK pages, not null), target_title, target_page_id (FK pages, nullable — for v0.3 graph resolution), alias (null), dangling (default false), created_at.

    provider_name and model_id in ingest_runs are audit metadata only — never read back into a routing decision (I6).

    6. MCP tool contracts (backend/app/mcp/server.py, FastMCP, stdio)¶

    All four tools reuse the ingest seam primitives (ADR-0010 §2): write_page does NOT write files directly — it validates, slugs, writes, then calls persist_metadata/upsert_vector/append_log/bump_version + K5 parse + K3 index, the same primitives the orchestrator uses (I1, I5). search_wiki reuses Qdrant + bge-m3 (I9).

    Tool Input Output Notes
    search_wiki query: str list[PageRef] where PageRef = vector lookup on synapse_pages (I9); NOT F5 RAG
    write_page title: str, content: str, frontmatter: dict PageRef or structured tool-error validates type/title/sources (ADR-0007 §5); error (not exception) on missing field; updates index.md
    get_page title: str {title, type, content, frontmatter} or not-found tool-error structured error on unknown title
    list_pages type: str \| null list[PageRef] excludes soft-deleted (deleted_at != NULL); optional type filter

    Schemas exported to docs/api/mcp-tools.json (AC-MCP-7/AC-D4u-2). CliAgentProvider consumes these over stdio during delegated ingest (AC-MCP-8).

    7. Invariant-compliance statement¶

    Invariant How the v0.2 design honours it
    I6 (pluggable, never hardcode) All inference behind the ABC; routing reads capabilities().supports_agentic_loop only (no isinstance/class/type — AC-K2-4); provider selected from provider_config (operation>vault>global); model ids only in seeded DB rows (AC-F17-8); keys env-only, confined to provider/ (AC-F17-4); provider_name stored for audit, never routed on. Backends (incl. OpenAI-compat) are config (base_url), not new classes.
    I7 (bounded loops, cost logged) Orchestrated loop bounded by max_iter=3 AND token_budget=60k (dual stop, pre-call budget check); CLI budget=100k passed to SDK; fallback bounded to exactly 1; every run writes ingest_runs with total_cost_usd (0.00 for local/cli) + converged; inline $1 anomaly WARNING. Non-convergence test (AC-K2-5) stops at exactly 3.
    I1 (incremental only) write_page (both orchestrated and MCP) calls the same upsert primitives; K3 index + K5 links update per-write event; no rescan.
    I5 (Obsidian valid) Typed WikiFrontmatter enforces type/title/sources/lang at the schema boundary; validator rejects pages lacking them before write; index.md/overview.md keep valid frontmatter. Human re-check at EC-M2-13.
    I8 (docs-as-DoD) ADR-0007..0011 authored; component.mmd added, context/container updated; D3 ingest-routing.mmd + ingest-loop.mmd drafted; D2 to be regenerated via make er after models.py lands; D4 OpenAPI + mcp-tools.json.
    I9 (do not reinvent) OllamaProvider reuses running Ollama; search_wiki reuses Qdrant/bge-m3; FastMCP for the server; no new inference/search container.
    I2/I3/I4 Not exercised in v0.2 (no graph/chat-UI/editor); design introduces nothing that breaks them.

    EC coverage flag¶

    The design satisfies all of EC-M2-1 through EC-M2-16 at the design level. The following are execution/runtime gates the design cannot itself satisfy — they require live infra and the human checkpoint, and are explicitly out of the architect's sign-off scope: - EC-M2-5 (3-provider live smoke matrix) and EC-M2-17 (human checkpoint) — require running Ollama + a real Anthropic key + claude-agent-sdk on TrueNAS. Design is complete and correct; execution is the qa/human gate. - EC-M2-13 (Obsidian open with real LLM content) — manual human verification.

    No EC is unsatisfiable by design. No invariant is traded.

    9. F17 abstraction — SIGN-OFF¶

    VERDICT: APPROVED to implement.

    The InferenceProvider ABC, capability-aware routing, bounded ingest loop, provider_config + ingest_runs + links data model, and MCP tool contracts are signed off as designed in this document and ADR-0007..0011. All 8 AQ questions are resolved. No invariant is traded for convenience; I6 and I7 are honoured with machine-checkable guardrails (static grep for provider leakage and routing-by-type; the non-converging-loop test).

    Conditions on merge (architect review gate EC-M2-15 will verify): 1. orchestrator.py routing branch contains no isinstance/type/class-name test (CI grep). 2. No model id / key / endpoint outside provider/ (CI grep, AC-F17-4/8). 3. validate() and write_page share the ONE validator (ADR-0007 §5 / ADR-0010 §2). 4. The non-converging loop stops at exactly max_iter=3 with converged=False (AC-K2-5). 5. D2 regenerated by make er matches the live schema after the migration lands.
