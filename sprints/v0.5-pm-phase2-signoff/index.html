    FU-P2-2: No-provider mechanical-completion path (architectural note for USER.md)¶

    The architect noted that when no provider_config row exists for a vault, _resolve_provider returns None. The loop degrades gracefully: query generation falls back to the raw topic as the single query; sufficiency assessment treats any content as sufficient; synthesis assembles raw snippets. The run completes bounded, SearXNG-only, via the ingest seam — all invariants are held. However, the resulting wiki page is a low-quality stub with no LLM processing.

    This is invariant-safe but counter-intuitive to users. It must be documented in USER.md (D6a) in the F10 / Deep Research section: "If no inference provider is configured for the vault, Deep Research completes as a raw-fetch stub with no LLM query refinement or synthesis. Configure a provider in Settings > LLM Models before running Deep Research." Owner: tech-writer (D6a update, to be folded into Phase 4 final docs gate or Phase 5).

    FU-P2-3: $1 cost-anomaly WARNING test gap (accepted, non-blocking)¶

    No dedicated unit test asserts that logger.warning(...) fires when total_cost_usd > 1.00. The module-level constant COST_ANOMALY_THRESHOLD_USD = 1.00 exists; the code path fires in the finally block; the threshold is structurally verified. The anomaly is a logging/observability feature — it cannot cause runaway execution and does not violate I7. QA accepted this gap as belt-and-suspenders. It may be added as an optional enhancement (a caplog fixture test) in Phase 3 if a QA cycle permits, but it does not hold Phase 2 or Phase 3. Resolution: log as GAP-v0.5-P2-1 in the gap register; revisit at M5 final gate.
