    Sprint 2 — v0.2 (M2) — Architect PR Review + F17 Invariant + Diagram Sign-off¶

    Reviewer: solution-architect Date: 2026-06-28 Milestone: M2 — "Agentic loop closed, 3 providers" Gate: EC-M2-15 (architect sign-off on D3 + D7 + D1u); the I6/I7 invariant gate Scope: implementation review of F17 against docs/sprints/v0.2-architecture.md + ADR-0007..0011, and the 9 invariants in CLAUDE.md §3. Code was NOT modified. Authority: this verdict gates the merge of sprint/v0.2.

    This review verifies the IMPLEMENTATION against the design I signed off in v0.2-architecture.md §9. It is a per-item APPROVE/BLOCK gate. Verdict at §10.

    1. I6 — Pluggable inference (capability-aware routing, no hardcode) — APPROVE¶

    Check Evidence Result
    Routing decides on capabilities().supports_agentic_loop ONLY orchestrator.run_ingest_pipeline() reads caps = provider.capabilities() then if caps.supports_agentic_loop: (orchestrator.py:313/322). No other signal gates the branch. PASS
    No isinstance / type() / class-name / provider_type == in the routing path grep of orchestrator.py + loop.py: the only isinstance hits are unrelated (page.type PageType check in the validator, _sources_raw list coercion). No type(, __class__, provider_type ==, or .name == branch anywhere in routing. PASS
    resolve_provider has NO hardcoded default backend, raises on unknown/missing type provider/__init__.py:68 — None row → ValueError ("no hardcoded default"); unknown provider_type → ValueError listing valid types. _REGISTRY is a config-value→class dispatch (the row's provider_type), explicitly NOT a routing decision. PASS
    Resolution order operation > vault > global provider_config_service.resolve_provider_config() queries scope='operation'+vault+op, then scope='vault'+vault, then scope='global', each with is_fallback IS FALSE. ConfigNotFoundError on no match — never a silent default (I6). PASS
    No model id / API key / base_url literal outside provider_config rows + env grep across app/: the only claude-* strings are in column/field doc comments (models.py:264, main.py:207 descriptions) and the Alembic seed default (os.environ.get("DEFAULT_MODEL_ID", "claude-sonnet-4-6")). The only base_url literal is https://api.anthropic.com, confined to provider/api.py and env-overridable (ANTHROPIC_BASE_URL). No key literals; all keys via os.environ inside api.py/cli.py only. PASS
    chat() is a locked stub (NotImplementedError) on all 3 backends ollama.py:90, api.py:120, cli.py:97 all raise NotImplementedError(... v0.4 (F6)) with no network call. Signature locked per ABC. PASS
    Provider selectable per operation/vault _resolve_ingest_provider_config() resolves operation='ingest'; same ConfigResolver serves chat/lint scopes. Per-vault + per-operation rows honoured. PASS
    Backends are config, not new classes OpenAI-compatible path is base_url-driven inside the single ApiProvider (api.py:83 _openai_compatible = bool(config.base_url)). No subclass-per-vendor. PASS
    provider_name/model_id audit-only, never routed on Stored on ingest_runs from caps.name/config; never read back into a routing decision. PASS

    I6 VERDICT: APPROVE. Routing is single-signal capability-based with machine-checkable guardrails. No backend is hardcoded; no provider literal escapes provider/ + env + seeded DB rows. The CLI delegated vs orchestrated separation is correct (delegate_ingest on the agentic path; analyze()/generate() raise RuntimeError on CliAgentProvider to make accidental orchestrated use loud — exceeds the design ask).

    2. I7 — Bounded loops + cost logging — APPROVE¶

    Check Evidence Result
    Orchestrated loop bounded by BOTH max_iter AND token_budget loop.run_orchestrated_loop(): for i in range(1, max_iter+1) (bound #1) with a pre-call if accumulator.total_tokens >= token_budget: break (bound #2). Both read from provider_config with the ADR-0009 defaults (3 / 60 000). PASS
    analyze() runs ONCE; only generate() retries (AQ-v0.2-1) analyze() is called before the loop (loop.py:132); only generate() is inside the loop with augmented context. Matches ADR-0007 §4. PASS
    Non-convergence stops at max_iter, no overrun Loop exits cleanly with converged=False, stop_reason="max_iter"; iterations capped at max_iter. AC-K2-5 honoured. PASS
    token_budget halts before an unaffordable call Pre-call check sets stop_reason="token_budget" and breaks before generate(). Correct (analyze already spent budget). PASS
    total_cost_usd written to ingest_runs every run run_ingest_pipeline() finalizes the accumulator and always INSERT ingest_runs{...total_cost_usd...} on both routes (orchestrated and delegated). Numeric(10,4). PASS
    Fallback bounded to exactly once _run_orchestrated() catches (TimeoutError, ConnectionError), resolves ONE is_fallback=True row, retries once; a second failure raises IngestError with no chain (from exc2, no recursion). PASS
    $1/run anomaly → WARNING + cost_anomaly flag cost_anomaly = total_cost_usd > COST_ANOMALY_THRESHOLD_USD (1.00); flag persisted on the row; inline logger.warning(...) emitted AFTER the row write (AQ-v0.2-8). PASS
    Usage normalization across backends Ollama→prompt_eval_count/eval_count, cost 0.0; Anthropic→usage.input/output_tokens × env price map; OpenAI→prompt/completion_tokens; CLI→SDK metadata, cost 0.0 by convention with tokens=0 + WARNING when absent. All push via _record_usage to the run-scoped UsageAccumulator (out-of-band, ADR-0007 §1). PASS
    CLI budget passed/abort CliAgentProvider carries config.token_budget (100k default via DEFAULT_TOKEN_BUDGET_CLI); records exactly one ingest_runs row at the orchestrator. PASS

    I7 VERDICT: APPROVE. Dual bounds enforced together; analyze-once retry-generate is exact; fallback is exactly-once with no chaining; cost is persisted and anomaly-flagged on every run.

    Non-blocking note NB-1 (tracked, not a gate): the fallback except clause catches (TimeoutError, ConnectionError). ADR-0009 §4 phrases the trigger as "timeout / 503". An HTTP 503 surfaces from httpx as HTTPStatusError (via raise_for_status()), which is NOT a subclass of those two — so a literal 503 would currently surface as IngestError WITHOUT attempting the single fallback. This does not violate I7 (the loop and fallback remain bounded; no overrun, no chain) and the timeout path — the primary failure mode — is covered. Recommend a follow-up (v0.3 hardening) widening the catch to include httpx.HTTPStatusError for 5xx so the fallback engages on 503 as the ADR intends. Filed to backend-engineer; does not block M2.

    10. ARCHITECT VERDICT¶

    APPROVED.

    The F17 implementation conforms to the design signed off in v0.2-architecture.md §9 and honours all nine invariants. I6 (single-signal capability routing, no hardcoded backend, no provider-literal leakage, env-only keys, locked chat() stubs) and I7 (dual-bounded loop, analyze-once retry-generate, exactly-once fallback, per-run cost persistence + anomaly flag) — the two gate invariants for M2 — are met with machine-checkable guardrails. Data model, migration, and ER are consistent and drift-free. K5/K3 and the MCP write path correctly reuse the single write_wiki_page seam, preserving I1/I5; query_points is correct for qdrant-client ≥1.10. All D1/D2/D3/D7 diagrams are v0.2-accurate and signed off (EC-M2-15 cleared).

    This sign-off covers the design+implementation+docs gate. The remaining M2 items — EC-M2-5 (live 3-provider smoke on TrueNAS) and EC-M2-13/EC-M2-17 (open-in-Obsidian + human checkpoint) — are runtime/human gates outside the architect's scope and remain correctly deferred per the QA report.

    Hand to orchestrator: M2 architect gate MET (pending the documented live-smoke + human checkpoint). Three non-blocking follow-ups handed off: - NB-1 → backend-engineer: widen fallback catch to include httpx.HTTPStatusError 5xx (503). - DIAG-NB-1 → tech-writer: refresh component.mmd path labels (config/links/index modules). - (already tracked) openapi.json info.version/description string → backend-engineer.
